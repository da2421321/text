# 第二章：核心概念与工作原理

## 2.1 工具类优先（Utility-First）

TailwindCSS 的核心理念是将 CSS 属性拆分为单一职责的工具类。

### 传统方式 vs 工具类方式

```html
<!-- 传统：先写 HTML，再写 CSS -->
<div class="notification">
  <p class="notification-text">操作成功！</p>
</div>

<style>
.notification {
  padding: 12px 16px;
  background-color: #f0fdf4;
  border: 1px solid #86efac;
  border-radius: 8px;
}
.notification-text {
  color: #166534;
  font-size: 14px;
}
</style>

<!-- TailwindCSS：直接在 HTML 中组合 -->
<div class="px-4 py-3 bg-green-50 border border-green-300 rounded-lg">
  <p class="text-green-800 text-sm">操作成功！</p>
</div>
```

### 工具类命名规律

TailwindCSS 的工具类遵循统一的命名规律：

```
属性缩写 - 值
```

| 工具类 | 对应 CSS |
|--------|---------|
| `p-4` | `padding: 1rem` |
| `px-4` | `padding-left: 1rem; padding-right: 1rem` |
| `pt-2` | `padding-top: 0.5rem` |
| `m-auto` | `margin: auto` |
| `text-xl` | `font-size: 1.25rem; line-height: 1.75rem` |
| `font-bold` | `font-weight: 700` |
| `bg-blue-500` | `background-color: oklch(...)` |
| `rounded-lg` | `border-radius: 0.5rem` |
| `flex` | `display: flex` |
| `items-center` | `align-items: center` |

## 2.2 按需生成 CSS

TailwindCSS 只生成你实际使用的工具类，最终 CSS 文件极小。

### v4 自动内容检测

v4 不再需要配置 `content` 路径，它会自动扫描：
- 项目中所有 `.html`、`.js`、`.ts`、`.jsx`、`.tsx`、`.vue`、`.svelte` 等文件
- 排除 `node_modules`、`.git` 等目录

```css
/* 只需这一行，其余全自动 */
@import "tailwindcss";
```

### 手动指定扫描路径（`@source`）

如果需要扫描特殊路径，使用 `@source` 指令：

```css
@import "tailwindcss";

/* 扫描额外的目录 */
@source "../node_modules/@my-company/ui-kit/src";

/* 扫描特定文件 */
@source "./templates/**/*.html";
```

### 安全列表（强制包含某些类）

```css
@import "tailwindcss";

/* 强制包含这些类，即使代码中没有静态出现 */
@source inline("bg-red-500 bg-green-500 bg-blue-500");
```

## 2.3 `@import "tailwindcss"` 详解

这一行导入等价于 v3 的三行：

```css
/* v4 等价于 v3 的 */
@tailwind base;       /* 浏览器样式重置 */
@tailwind components; /* 组件层（默认为空） */
@tailwind utilities;  /* 工具类 */
```

### 分层导入（按需）

如果只需要部分层：

```css
/* 只导入工具类，不导入 base 重置 */
@import "tailwindcss/utilities";

/* 只导入 base 重置 */
@import "tailwindcss/base";

/* 只导入 preflight（浏览器重置） */
@import "tailwindcss/preflight";
```

## 2.4 任意值（Arbitrary Values）

当内置工具类不满足需求时，可以使用方括号语法传入任意值：

```html
<!-- 任意颜色 -->
<div class="bg-[#1da1f2]">Twitter 蓝</div>

<!-- 任意尺寸 -->
<div class="w-[320px] h-[180px]">固定尺寸</div>

<!-- 任意字体大小 -->
<p class="text-[13px]">自定义字号</p>

<!-- 任意 CSS 属性（CSS 变量） -->
<div class="bg-[var(--brand-color)]">使用 CSS 变量</div>

<!-- 任意 CSS 属性（方括号属性语法） -->
<div class="[mask-type:luminance]">任意 CSS 属性</div>
<div class="[--my-var:1rem]">设置 CSS 变量</div>
```

### 任意变体

```html
<!-- 任意选择器变体 -->
<div class="[&:nth-child(3)]:bg-blue-100">第三个子元素</div>
<div class="[&>p]:text-gray-600">直接子 p 元素</div>
<div class="[@media(min-width:900px)]:flex">自定义媒体查询</div>
```

## 2.5 CSS 变量与设计令牌

v4 的一大特性是所有设计令牌自动生成 CSS 变量。

```css
@import "tailwindcss";

@theme {
  --color-brand: #3B82F6;
  --spacing-18: 4.5rem;
  --font-size-2xs: 0.625rem;
}
```

生成后，你可以在任何地方使用这些 CSS 变量：

```css
.my-component {
  color: var(--color-brand);
  padding: var(--spacing-18);
}
```

同时，TailwindCSS 工具类也会自动支持：

```html
<div class="text-brand">品牌色文字</div>
<div class="p-18">自定义间距</div>
```

## 2.6 修饰符（Modifiers）

### 响应式修饰符

```html
<div class="text-sm md:text-base lg:text-lg">
  响应式文字大小
</div>
```

### 状态修饰符

```html
<button class="bg-blue-500 hover:bg-blue-600 active:bg-blue-700 focus:ring-2">
  按钮
</button>
```

### 修饰符叠加

```html
<!-- 在 md 断点下，hover 时改变颜色 -->
<div class="md:hover:bg-blue-100">
  组合修饰符
</div>
```

### 重要性修饰符（`!`）

```html
<!-- 添加 !important -->
<div class="!text-red-500">强制红色</div>
```

## 2.7 工具类分类速览

```
布局：      container flex grid block hidden overflow-hidden ...
Flexbox：   flex-row flex-col items-center justify-between gap-4 ...
Grid：      grid-cols-3 col-span-2 row-span-2 ...
间距：      p-4 px-6 py-2 m-auto mt-4 space-x-4 gap-6 ...
尺寸：      w-full h-screen max-w-lg min-h-0 ...
排版：      text-xl font-bold text-gray-900 leading-relaxed ...
背景：      bg-white bg-blue-500 bg-gradient-to-r ...
边框：      border border-gray-200 rounded-lg ring-2 ...
效果：      shadow-md opacity-50 blur-sm ...
过渡：      transition duration-200 ease-in-out ...
变换：      scale-105 rotate-45 translate-x-4 ...
```

## 小结

- 工具类优先：直接在 HTML 中组合原子化 CSS 类
- v4 自动检测内容，无需配置 `content` 路径
- `@import "tailwindcss"` 是新的导入语法
- 任意值语法 `[]` 处理特殊需求
- 所有设计令牌自动生成 CSS 变量
