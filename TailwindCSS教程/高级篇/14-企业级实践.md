# 第十四章：企业级实践

## 14.1 设计令牌管理

企业级项目需要统一管理设计令牌，确保视觉一致性。

### 令牌文件结构

```
src/
  styles/
    tokens/
      colors.css      # 颜色令牌
      typography.css  # 字体令牌
      spacing.css     # 间距令牌
      shadows.css     # 阴影令牌
    themes/
      light.css       # 亮色主题
      dark.css        # 深色主题
    main.css          # 入口文件
```

`src/styles/tokens/colors.css`：


## 14.2 多品牌主题

## 14.3 团队协作规范

### ESLint + Prettier 配置

```bash
npm install -D eslint-plugin-tailwindcss prettier-plugin-tailwindcss
```

`.eslintrc.cjs`：

```js
module.exports = {
  plugins: ['tailwindcss'],
  rules: {
    // 强制类名排序
    'tailwindcss/classnames-order': 'warn',
    // 禁止不必要的任意值
    'tailwindcss/no-arbitrary-value': 'off',
    // 禁止自定义类名与工具类混用
    'tailwindcss/no-custom-classname': 'warn',
  },
}
```

`.prettierrc`：

```json
{
  "plugins": ["prettier-plugin-tailwindcss"],
  "tailwindStylesheet": "./src/styles/main.css"
}
```

### 类名排序规范

推荐的类名书写顺序：

```html
<!-- 1. 布局 → 2. 尺寸 → 3. 间距 → 4. 排版 → 5. 颜色 → 6. 边框 → 7. 效果 → 8. 交互 -->
<div class="
  flex items-center justify-between
  w-full max-w-lg
  px-4 py-3
  text-sm font-medium
  bg-white text-gray-900
  border border-gray-200 rounded-xl
  shadow-sm
  hover:shadow-md transition-shadow
">
```

## 14.4 与 UI 组件库共存

### 与 Ant Design Vue 共存

```css
@import "tailwindcss";

/* 避免 Tailwind preflight 影响 Ant Design 样式 */
@import "tailwindcss/preflight" layer(tailwind-preflight);

/* 或者完全禁用 preflight */
@import "tailwindcss/utilities";
@import "tailwindcss/base" layer(tailwind-base);
```

更好的方案：给 TailwindCSS 添加前缀

```css
@import "tailwindcss" prefix(tw);
```

```html
<!-- 使用 tw- 前缀，避免与 Ant Design 冲突 -->
<div class="tw-flex tw-items-center tw-gap-4">
  <a-button type="primary">Ant Design 按钮</a-button>
  <span class="tw-text-gray-500">TailwindCSS 文字</span>
</div>
```

### 与 Element Plus 共存

```css
@import "tailwindcss";

/* 提高 TailwindCSS 优先级 */
@layer base, element-plus, tailwind-utilities;
```

## 14.5 代码审查清单

```markdown
## TailwindCSS 代码审查清单

### 可维护性
- [ ] 避免超过 10 个工具类的长类名，考虑提取为组件
- [ ] 响应式类名按断点顺序书写（sm: md: lg:）
- [ ] 深色模式类名紧跟对应的亮色类名

### 性能
- [ ] 没有动态拼接类名（如 `bg-${color}-500`）
- [ ] 动态类名使用对象映射
- [ ] 没有不必要的 `!important`（`!` 前缀）

### 一致性
- [ ] 颜色使用设计令牌，不使用任意值（除非必要）
- [ ] 间距使用标准值，不使用任意值
- [ ] 交互状态（hover/focus/active）完整

### 可访问性
- [ ] 按钮有 focus 样式（`focus:ring-*`）
- [ ] 颜色对比度满足 WCAG AA 标准
- [ ] 不仅依赖颜色传达信息
```

## 14.6 Monorepo 中的 TailwindCSS

```
packages/
  ui/                    # 共享 UI 组件库
    src/
      styles/
        tokens.css       # 共享设计令牌
    package.json
  web/                   # Web 应用
    src/
      styles/
        main.css
    vite.config.ts
  admin/                 # 管理后台
    src/
      styles/
        main.css
    vite.config.ts
```

`packages/ui/src/styles/tokens.css`：

```css
@theme {
  /* 共享设计令牌 */
  --color-brand-600: oklch(52% 0.22 250);
  --font-sans: 'Inter', sans-serif;
}
```

`packages/web/src/styles/main.css`：

```css
@import "tailwindcss";
@import "@company/ui/styles/tokens.css";

/* Web 应用特有令牌 */
@theme {
  --color-accent: oklch(65% 0.2 145);
}
```

## 14.7 CI/CD 集成

```yaml
# .github/workflows/build.yml
name: Build

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'

      - run: pnpm install

      # 检查类名排序
      - run: pnpm lint

      # 构建并检查 CSS 大小
      - run: pnpm build
      - name: Check CSS bundle size
        run: |
          CSS_SIZE=$(find dist -name "*.css" -exec wc -c {} + | tail -1 | awk '{print $1}')
          echo "CSS bundle size: ${CSS_SIZE} bytes"
          # 如果超过 100KB 则警告
          if [ $CSS_SIZE -gt 102400 ]; then
            echo "Warning: CSS bundle size exceeds 100KB"
          fi
```

## 小结

- 设计令牌分文件管理，通过 `@import` 组合
- 多品牌主题通过 CSS 变量覆盖实现
- `prettier-plugin-tailwindcss` 自动排序类名
- 与 UI 组件库共存：使用 `prefix(tw)` 或禁用 preflight
- Monorepo 中共享设计令牌包
