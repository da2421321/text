# 第十五章：v3 迁移指南

## 15.1 迁移概览

从 v3 迁移到 v4 的主要工作：

1. 更新依赖包
2. 替换 CSS 导入语法
3. 将 `tailwind.config.js` 转换为 `@theme` 指令
4. 处理破坏性变更

## 15.2 自动迁移工具

官方提供了自动迁移工具，可以处理大部分变更：

```bash
# 安装迁移工具
npx @tailwindcss/upgrade@next

# 或者指定项目目录
npx @tailwindcss/upgrade@next --input ./src/styles/main.css
```

迁移工具会自动：
- 更新 `package.json` 中的依赖
- 转换 CSS 导入语法
- 将 `tailwind.config.js` 转换为 `@theme` 指令
- 处理已知的破坏性变更

## 15.3 手动迁移步骤

### 第一步：更新依赖

```bash
# 移除旧依赖
npm uninstall tailwindcss autoprefixer

# 安装新依赖（Vite 项目）
npm install tailwindcss @tailwindcss/vite

# 或 PostCSS 项目
npm install tailwindcss @tailwindcss/postcss
```

### 第二步：更新构建配置

**Vite 项目**：

```js
// vite.config.js（v3）
import { defineConfig } from 'vite'
export default defineConfig({ /* ... */ })

// vite.config.js（v4）
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'
export default defineConfig({
  plugins: [tailwindcss()],
})
```

**PostCSS 项目**：

```js
// postcss.config.js（v3）
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}

// postcss.config.js（v4）
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

### 第三步：更新 CSS 导入

```css
/* v3 */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* v4 */
@import "tailwindcss";
```

### 第四步：迁移配置文件

**v3 `tailwind.config.js`**：

```js
module.exports = {
  content: ['./src/**/*.{html,js,vue,tsx}'],
  theme: {
    extend: {
      colors: {
        brand: {
          500: '#3B82F6',
          600: '#2563EB',
        },
      },
      fontFamily: {
        sans: ['Inter', 'sans-serif'],
      },
      spacing: {
        18: '4.5rem',
      },
      borderRadius: {
        '4xl': '2rem',
      },
    },
  },
  plugins: [],
}
```

**v4 CSS 配置**：

```css
@import "tailwindcss";

/* content 配置不再需要，自动检测 */

@theme {
  /* colors.brand → --color-brand-* */
  --color-brand-500: #3B82F6;
  --color-brand-600: #2563EB;

  /* fontFamily.sans → --font-sans */
  --font-sans: 'Inter', sans-serif;

  /* spacing.18 → --spacing-18 */
  --spacing-18: 4.5rem;

  /* borderRadius.4xl → --radius-4xl */
  --radius-4xl: 2rem;
}
```

## 15.4 破坏性变更清单

### 工具类重命名

| v3 | v4 |
|----|-----|
| `shadow-sm` | `shadow-xs` |
| `shadow` | `shadow-sm` |
| `shadow-md` | `shadow-md`（不变） |
| `blur-sm` | `blur-xs` |
| `blur` | `blur-sm` |
| `rounded` | `rounded-sm` |
| `rounded-sm` | `rounded-xs` |
| `ring` | `ring-3` |
| `ring-1` | `ring-1`（不变） |
| `outline` | `outline-1` |
| `border` | `border`（不变） |

### 颜色变化

v4 使用 OKLCH 色彩空间，颜色值有细微变化，但工具类名称不变。

### 默认值变化

```
v3 默认边框颜色：currentColor
v4 默认边框颜色：--color-gray-200

v3 默认 ring 颜色：--color-blue-500
v4 默认 ring 颜色：currentColor

v3 placeholder 颜色：--color-gray-400
v4 placeholder 颜色：color-mix(in oklab, currentColor 50%, transparent)
```

### 移除的功能

```
- tailwind.config.js（改用 @theme）
- @tailwind base/components/utilities（改用 @import "tailwindcss"）
- theme() CSS 函数（改用 CSS 变量 var(--*)）
- screen() CSS 函数（改用 @media (width >= theme(--breakpoint-md))）
- @layer components/utilities（改用 @utility 或直接写 CSS）
```

## 15.5 常见迁移问题

### 问题 1：`theme()` 函数不再支持

```css
/* v3 */
.my-class {
  color: theme('colors.blue.500');
  padding: theme('spacing.4');
}

/* v4 */
.my-class {
  color: var(--color-blue-500);
  padding: var(--spacing-4);
}
```

### 问题 2：`@layer utilities` 改为 `@utility`

```css
/* v3 */
@layer utilities {
  .scrollbar-hide {
    scrollbar-width: none;
  }
}

/* v4 */
@utility scrollbar-hide {
  scrollbar-width: none;
}
```

### 问题 3：`@layer components` 改为直接写 CSS

```css
/* v3 */
@layer components {
  .btn {
    @apply px-4 py-2 rounded-lg font-semibold;
  }
}

/* v4 */
.btn {
  @apply px-4 py-2 rounded-lg font-semibold;
}
```

### 问题 4：插件 API 变化

```js
// v3 插件
const plugin = require('tailwindcss/plugin')
module.exports = plugin(function({ addUtilities }) {
  addUtilities({ '.my-util': { display: 'flex' } })
})

// v4 插件（ES Module）
import plugin from 'tailwindcss/plugin'
export default plugin(function({ addUtilities }) {
  addUtilities({ '.my-util': { display: 'flex' } })
})
```

### 问题 5：JIT 模式（v4 默认开启，无需配置）

v3 需要配置 `mode: 'jit'`，v4 默认就是 JIT 模式，无需任何配置。

## 15.6 渐进式迁移策略

对于大型项目，建议渐进式迁移：

```
阶段 1：更新依赖和构建配置（1天）
  - 安装 v4 依赖
  - 更新 vite.config.js / postcss.config.js
  - 更新 CSS 导入语法

阶段 2：迁移配置（1-2天）
  - 将 tailwind.config.js 转换为 @theme
  - 运行自动迁移工具

阶段 3：处理破坏性变更（1-3天）
  - 修复工具类重命名
  - 替换 theme() 函数为 CSS 变量
  - 更新自定义插件

阶段 4：优化（可选）
  - 使用新的 @utility 指令
  - 使用新的变体（not-*、nth-* 等）
  - 利用 CSS 变量简化主题切换
```

## 15.7 迁移验证

```bash
# 构建并检查是否有错误
npm run build

# 检查是否有遗漏的 theme() 函数
grep -r "theme(" src/

# 检查是否有旧的导入语法
grep -r "@tailwind" src/

# 检查是否有旧的配置文件
ls tailwind.config.*
```

## 小结

- 使用官方迁移工具 `@tailwindcss/upgrade` 自动处理大部分变更
- 主要变化：CSS 导入语法、配置方式、部分工具类重命名
- `theme()` 函数改为 CSS 变量 `var(--*)`
- `@layer utilities` 改为 `@utility`
- 大型项目建议分阶段渐进式迁移
