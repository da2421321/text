# 第十二章：性能优化

## 12.1 TailwindCSS v4 的性能优势

v4 基于 Rust 的 Oxide 引擎，性能大幅提升：

| 场景 | v3 | v4 |
|------|----|----|
| 全量构建 | ~3.5s | ~0.4s |
| 增量构建 | ~350ms | ~4ms |
| 内存占用 | 较高 | 降低约 35% |

## 12.2 生产构建优化

### Vite 生产构建

```js
// vite.config.js
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
  build: {
    // CSS 代码分割
    cssCodeSplit: true,
    // 压缩
    minify: 'esbuild',
    rollupOptions: {
      output: {
        // 分离 CSS
        assetFileNames: 'assets/[name]-[hash][extname]',
      },
    },
  },
})
```

### 分析 CSS 产物大小

```bash
# 构建并分析
npm run build

# 查看 CSS 文件大小
ls -lh dist/assets/*.css
```

典型的生产 CSS 大小：
- 小型项目：5-15 KB（gzip 后 2-5 KB）
- 中型项目：15-40 KB（gzip 后 5-12 KB）
- 大型项目：40-80 KB（gzip 后 12-25 KB）

## 12.3 避免动态类名

TailwindCSS 通过静态扫描生成 CSS，动态拼接类名会导致样式丢失：

```js
// ❌ 错误：动态拼接，无法被扫描到
const color = 'blue'
const className = `bg-${color}-500`  // bg-blue-500 不会被生成

// ✅ 正确：完整类名
const classMap = {
  blue: 'bg-blue-500',
  red: 'bg-red-500',
  green: 'bg-green-500',
}
const className = classMap[color]
```

```vue
<!-- ❌ 错误 -->
<div :class="`text-${size === 'lg' ? 'xl' : 'base'}`">

<!-- ✅ 正确 -->
<div :class="size === 'lg' ? 'text-xl' : 'text-base'">
```

## 12.4 `@source` 精确控制扫描范围

```css
@import "tailwindcss";

/* 明确指定需要扫描的路径 */
@source "./src/**/*.{vue,ts,tsx}";
@source "./index.html";

/* 排除不需要扫描的路径（通过 not 语法） */
@source not "./src/legacy/**";
```

## 12.5 安全列表（Safelist）

对于动态生成的类名，使用安全列表强制包含：

```css
@import "tailwindcss";

/* 强制包含特定类 */
@source inline("
  bg-red-500 bg-orange-500 bg-yellow-500 bg-green-500 bg-blue-500 bg-purple-500
  text-red-500 text-orange-500 text-yellow-500 text-green-500 text-blue-500 text-purple-500
");
```

## 12.6 关键 CSS 提取

对于首屏性能要求高的场景，可以提取关键 CSS 内联到 HTML：

```js
// vite.config.js（使用 vite-plugin-critical）
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'
import critical from 'vite-plugin-critical'

export default defineConfig({
  plugins: [
    tailwindcss(),
    critical({
      criticalUrl: 'http://localhost:5173',
      criticalBase: './dist',
      criticalPages: [{ uri: '/', template: 'index' }],
      criticalConfig: { inline: true },
    }),
  ],
})
```

## 12.7 字体优化

```html
<!-- 预加载关键字体 -->
<link rel="preload" href="/fonts/inter-var.woff2" as="font" type="font/woff2" crossorigin>

<!-- 使用 font-display: swap 避免 FOIT -->
<style>
@font-face {
  font-family: 'Inter';
  src: url('/fonts/inter-var.woff2') format('woff2');
  font-display: swap;
}
</style>
```

```css
@import "tailwindcss";

@theme {
  /* 使用系统字体栈作为回退 */
  --font-sans: 'Inter', ui-sans-serif, system-ui, -apple-system, sans-serif;
}
```

## 12.8 图片与媒体优化

```html
<!-- 响应式图片 -->
<img
  src="image-800.jpg"
  srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
  sizes="(max-width: 640px) 100vw, (max-width: 1024px) 50vw, 33vw"
  class="w-full h-auto object-cover"
  loading="lazy"
  decoding="async"
  alt="图片描述"
>

<!-- 固定宽高比容器 -->
<div class="aspect-video overflow-hidden rounded-xl">
  <img src="video-thumbnail.jpg" class="w-full h-full object-cover" alt="">
</div>
```

## 12.9 减少重绘与回流

```html
<!-- 使用 transform 代替改变 top/left -->
<!-- ❌ 会触发回流 -->
<div class="hover:top-[-4px] relative transition-all">卡片</div>

<!-- ✅ 只触发合成层，性能更好 -->
<div class="hover:-translate-y-1 transition-transform">卡片</div>

<!-- 使用 will-change 提示浏览器 -->
<div class="will-change-transform hover:scale-105 transition-transform">
  频繁动画的元素
</div>
```

## 12.10 虚拟滚动与懒加载

对于长列表，配合虚拟滚动库使用：

```vue
<script setup>
import { useVirtualList } from '@vueuse/core'

const { list, containerProps, wrapperProps } = useVirtualList(items, {
  itemHeight: 80,
})
</script>

<template>
  <!-- 容器固定高度，内部虚拟滚动 -->
  <div v-bind="containerProps" class="h-[600px] overflow-auto scrollbar-hide">
    <div v-bind="wrapperProps">
      <div
        v-for="{ data: item } in list"
        :key="item.id"
        class="flex items-center gap-4 px-4 py-3 border-b border-gray-100 hover:bg-gray-50 transition-colors"
      >
        {{ item.name }}
      </div>
    </div>
  </div>
</template>
```

## 12.11 生产环境检查清单

```
✅ 使用 Vite 或 PostCSS 构建（不用 CDN）
✅ 开启 CSS 压缩（minify）
✅ 避免动态拼接类名
✅ 使用完整类名或对象映射
✅ 图片使用 lazy loading
✅ 字体使用 font-display: swap
✅ 关键路径 CSS 内联（可选）
✅ 开启 Brotli/Gzip 压缩（服务器配置）
✅ 使用 CDN 分发静态资源
```

## 小结

- v4 Oxide 引擎大幅提升构建速度
- 避免动态拼接类名，使用完整类名或对象映射
- `@source` 精确控制扫描范围
- `@source inline()` 安全列表处理动态类名
- 使用 `transform` 代替位置属性，减少回流
