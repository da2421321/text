# 02 - 配置管理与多环境 (Viper)

无论是前端的 `.env.development` 还是后端的配置文件，在企业级工程中，硬编码各种数据库密码或端口号是大忌。Go 语言生态中最著名、最被广泛使用的配置管理库是 [Viper](https://github.com/spf13/viper)。

## 引入 Viper

```bash
go get github.com/spf13/viper
```

## 1. 准备一份 YAML 配置文件
新建 `config/config.yaml`:
```yaml
server:
  port: 8080
  mode: debug

database:
  driver: mysql
  dsn: "user:pass@tcp(127.0.0.1:3306)/dbname?charset=utf8mb4&parseTime=True&loc=Local"

redis:
  host: 127.0.0.1
  port: 6379
  password: ""
  db: 0
```

## 2. 映射配置到结构体 (Struct)
利用你在基础篇学到的知识，我们需要把这个 yaml 映射成 Go 语言里的**结构体**。对于复杂的配置嵌套，我们就嵌套结构体：

在 `internal/config/config.go` 中定义：
```go
package config

import (
	"fmt"
	"github.com/spf13/viper"
	"os"
)

// 定义总配置结构体
type AppConfig struct {
	Server   ServerConfig   `mapstructure:"server"`
	Database DatabaseConfig `mapstructure:"database"`
	Redis    RedisConfig    `mapstructure:"redis"`
}

type ServerConfig struct {
	Port int    `mapstructure:"port"`
	Mode string `mapstructure:"mode"`
}

type DatabaseConfig struct {
	Driver string `mapstructure:"driver"`
	Dsn    string `mapstructure:"dsn"`
}

type RedisConfig struct {
	Host     string `mapstructure:"host"`
	Port     int    `mapstructure:"port"`
	Password string `mapstructure:"password"`
	DB       int    `mapstructure:"db"`
}

// 声明一个全局变量保存配置（通常企业级项目里推荐单例或依赖注入传递）
var Conf = new(AppConfig)

// 初始化配置
func InitConfig() error {
	// 支持按环境加载：APP_ENV=dev/test/prod
	env := os.Getenv("APP_ENV")
	if env == "" {
		env = "dev"
	}
	viper.SetConfigFile("config/config." + env + ".yaml")

	// 允许环境变量覆盖（示例：SERVER_PORT=9090）
	viper.AutomaticEnv()
	
	// 读取配置信息
	if err := viper.ReadInConfig(); err != nil {
		return err
	}
	
	// 将读取到的配置信息反序列化解析到 Conf 变量中去
	if err := viper.Unmarshal(Conf); err != nil {
		return err
	}

	// 最小化配置校验，启动前尽早失败
	if Conf.Server.Port <= 0 {
		return fmt.Errorf("server.port 配置非法")
	}
	if Conf.Database.Dsn == "" {
		return fmt.Errorf("database.dsn 不能为空")
	}
	return nil
}
```

## 3. 在 main 中初始化加载

在 `cmd/server/main.go` 中：

```go
package main

import (
	"fmt"
	"my-go-backend/internal/config"
)

func main() {
	// 1. 系统启动时第一步就是加载配置
	if err := config.InitConfig(); err != nil {
		panic(fmt.Sprintf("加载配置失败: %v", err))
	}

	fmt.Printf("服务跑在端口: %d\n", config.Conf.Server.Port)
	fmt.Printf("准备连接数据库: %s\n", config.Conf.Database.Dsn)

    // 然后再启动 Gin 服务...
}
```

通过这种方式，更换环境（如读取 `config/config.prod.yaml`）就非常简单，不需要在业务代码里改任何配置字符串。

## 4. 建议的多环境文件命名

- `config/config.dev.yaml`
- `config/config.test.yaml`
- `config/config.prod.yaml`

通过 `APP_ENV=prod` 进行切换，部署脚本只改环境变量即可。
