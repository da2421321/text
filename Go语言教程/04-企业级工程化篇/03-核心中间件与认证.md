# 03 - 核心中间件与鉴权认证 (Middleware & JWT)

在 Web 开发中，**中间件 (Middleware)** 提供了一种面向切面编程 (AOP) 的方式，用来在请求真正到达业务逻辑之前（或之后）进行拦截处理。比如跨域处理 (CORS) 和大名鼎鼎的 JWT 令牌鉴权。

Gin 框架对中间件的支持极为直观，类似 Express 的 `app.use(xxxxx)`。

## 1. CORS 跨域中间件

由于我们的前端常常运行在 `localhost:5173` 等端口，跨域问题不可避免。在 Go 后端通过中间件统一添加跨域 Header：

在 `internal/middleware/cors.go` 中：
```go
package middleware

import (
	"github.com/gin-gonic/gin"
	"net/http"
)

func Cors() gin.HandlerFunc {
	return func(c *gin.Context) {
		method := c.Request.Method
		origin := c.GetHeader("Origin")
		allowed := map[string]bool{
			"http://localhost:5173": true,
			"https://your-frontend.com": true,
		}
		// 注意：Allow-Credentials=true 时，Allow-Origin 不能是 "*"
		if allowed[origin] {
			c.Header("Access-Control-Allow-Origin", origin)
		}
		c.Header("Access-Control-Allow-Methods", "POST, GET, OPTIONS, PUT, DELETE, UPDATE")
		c.Header("Access-Control-Allow-Headers", "Content-Type, AccessToken, X-CSRF-Token, Authorization, Token")
		c.Header("Access-Control-Expose-Headers", "Content-Length, Access-Control-Allow-Origin, Access-Control-Allow-Headers, Content-Type")
		c.Header("Access-Control-Allow-Credentials", "true")
		c.Header("Vary", "Origin")

		// 浏览器对于复杂请求会发一个 OPTIONS 嗅探请求，直接放行返回 204
		if method == "OPTIONS" {
			c.AbortWithStatus(http.StatusNoContent)
		}
		// next() 的概念：放行让后面的函数或下一步路由执行
		c.Next()
	}
}
```

## 2. JWT 登录鉴权中间件

几乎所有的 To-C 或 To-B 项目都需要登录会话保持，而现在主流做法是使用 JWT (JSON Web Token)。
当用户拿到 Token 后，在 Header 中带上 `Authorization: Bearer <token>` 发起请求。

在 Go 中我们需要用到 JWT 库：`go get github.com/golang-jwt/jwt/v5`

首先是前置条件（生成和解析 token 工具库在此省略），我们来看鉴权拦截器本身该怎么写。

在 `internal/pkg/jwt/jwt.go` 中，可先准备一个最小实现：

```go
package jwtpkg

import (
	"fmt"
	"github.com/golang-jwt/jwt/v5"
	"time"
)

var secret = []byte("replace-with-env-secret")

type MyClaims struct {
	UserID uint `json:"user_id"`
	jwt.RegisteredClaims
}

func GenerateToken(userID uint) (string, error) {
	claims := MyClaims{
		UserID: userID,
		RegisteredClaims: jwt.RegisteredClaims{
			ExpiresAt: jwt.NewNumericDate(time.Now().Add(2 * time.Hour)),
			IssuedAt:  jwt.NewNumericDate(time.Now()),
		},
	}
	token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
	return token.SignedString(secret)
}

func ParseToken(tokenString string) (*MyClaims, error) {
	token, err := jwt.ParseWithClaims(tokenString, &MyClaims{}, func(token *jwt.Token) (interface{}, error) {
		if token.Method != jwt.SigningMethodHS256 {
			return nil, fmt.Errorf("unexpected signing method")
		}
		return secret, nil
	})
	if err != nil {
		return nil, err
	}
	claims, ok := token.Claims.(*MyClaims)
	if !ok || !token.Valid {
		return nil, fmt.Errorf("invalid token")
	}
	return claims, nil
}
```

在 `internal/middleware/jwt_auth.go` 中：
```go
package middleware

import (
	"github.com/gin-gonic/gin"
	"net/http"
	"strings"
)

func JWTAuth() gin.HandlerFunc {
	return func(c *gin.Context) {
		// 1. 获取请求头里的 token (格式：Bearer xxxxxx)
		authHeader := c.Request.Header.Get("Authorization")
		if authHeader == "" {
			c.JSON(http.StatusUnauthorized, gin.H{"code": 401, "msg": "请求头中auth为空"})
			c.Abort() // 终止往下传递到 controller 
			return
		}

		parts := strings.SplitN(authHeader, " ", 2)
		if !(len(parts) == 2 && parts[0] == "Bearer") {
			c.JSON(http.StatusUnauthorized, gin.H{"code": 401, "msg": "请求头中auth格式有误"})
			c.Abort()
			return
		}

		// 2. 解析和校验 JWT Token 的合法性
		// 假设 myJwtPkg.ParseToken(parts[1]) 是我们封装好的 JWT 验证函数
		claims, err := jwtpkg.ParseToken(parts[1])
		if err != nil {
			c.JSON(http.StatusUnauthorized, gin.H{"code": 401, "msg": "无效的Token"})
			c.Abort()
			return
		}

		// 3. 将解析出来的用户信息 (如 ID) 挂载到 Context 下下文中！
		// 这样 controller 里就可以直接用 c.Get("userID") 拿到当前是哪个用户发起的请求了
		c.Set("userID", claims.UserID)
		
		c.Next() // 放行
	}
}
```

## 3. 在路由上注册使用 

```go
func main() {
    r := gin.Default()

    // 注册全局跨域中间件
    r.Use(middleware.Cors())

    // 无需鉴权的公开路由
    r.POST("/api/login", userController.Login)

    // 需要鉴权的非公开路由的分组
    authGroup := r.Group("/api/admin")
    // 为这个组别挂载 JWTAuth 中间件
    authGroup.Use(middleware.JWTAuth())
    {
        authGroup.GET("/profile", userController.GetProfile) // 请求进入这里前必须通过 JWTAuth
        authGroup.POST("/settings", userController.UpdateSettings)
    }

    r.Run(":8080")
}
```

学会灵活运用 `c.Abort()` (拦截打回) 和 `c.Next()` (放行) ，你就掌握了 Web 服务进阶的门钥匙。

## 4. 鉴权实践补充

- `JWT secret` 不要硬编码，放入环境变量或配置中心。
- Access Token 建议短时效（如 15 分钟到 2 小时），配合 Refresh Token 续期。
- 对登录、刷新 token 接口加限流，防止撞库与暴力攻击。
