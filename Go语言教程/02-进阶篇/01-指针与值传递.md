# 进阶一：指针与值传递（深入内存思维）

> 本章目标：彻底理解 Go 的值传递机制和指针，知道什么时候该用指针、什么时候用值。

## 前置知识

学习本章前，你需要掌握：
- 变量与数据类型（基础篇第二章）
- 函数（基础篇第四章）
- 结构体（基础篇第七章）

## 1. 从一个困惑开始

先看一个让很多新手困惑的例子：

```go
package main

import "fmt"

type User struct {
    Name string
    Age  int
}

func birthday(u User) {
    u.Age++
    fmt.Println("函数内部：", u.Age) // 26
}

func main() {
    user := User{Name: "张三", Age: 25}
    birthday(user)
    fmt.Println("函数外部：", user.Age) // 25 ← 为什么没变？
}
```

调用了 `birthday` 之后，`Age` 居然没变？这就涉及到 Go 最核心的概念之一：**值传递**。

## 2. Go 永远是值传递

**值传递**的意思是：当你把变量传给函数时，Go 会**复制一份**给函数。函数里操作的是副本，不影响原始变量。

```
调用前：
  main 的 user → { Name:"张三", Age:25 }

调用时（传给 birthday）：
  main 的 user → { Name:"张三", Age:25 }  ← 原始数据
  函数的 u     → { Name:"张三", Age:25 }  ← 复制的副本

函数内 u.Age++：
  main 的 user → { Name:"张三", Age:25 }  ← 没变
  函数的 u     → { Name:"张三", Age:26 }  ← 只有副本变了
```

这就像你把一份文件**复印**一份给同事，同事在复印件上写字，不会影响你手上的原件。

## 3. 指针：传递"地址"

如果你想让函数能修改原始数据，需要传递**指针**——即变量在内存中的"地址"。

### 什么是指针？

想象内存是一排带编号的柜子：

```
地址：  0x001    0x002    0x003    0x004
       ┌───────┬────────┬────────┬────────┐
       │ "张三" │  25    │ "李四" │  30    │
       └───────┴────────┴────────┴────────┘
```

- **变量**存放的是"值"（柜子里的东西）
- **指针**存放的是"地址"（柜子的编号）

### 指针的语法

```go
name := "张三"

// & 取地址：获取变量的内存地址
p := &name
fmt.Println(p)  // 0xc000010200（一个内存地址）

// * 解引用：通过地址访问/修改值
fmt.Println(*p) // 张三（取出地址指向的值）

*p = "李四"      // 通过指针修改原始变量的值
fmt.Println(name) // 李四（原始变量也变了！）
```

两个关键符号：
- `&变量` → 取变量的地址（"这个柜子的编号是多少？"）
- `*指针` → 取指针指向的值（"这个编号的柜子里有什么？"）

### 指针类型

```go
var name string = "张三"
var p *string = &name  // *string 表示"指向 string 的指针"
fmt.Println("name 的值:", name)        // name 的值: 张三
fmt.Println("p 的地址:", p)            // p 的地址: 0xc000010200
fmt.Println("p 指向的值:", *p)         // p 指向的值: 张三

var age int = 25
var q *int = &age      // *int 表示"指向 int 的指针"
fmt.Println("age 的值:", age)          // age 的值: 25
fmt.Println("q 的地址:", q)            // q 的地址: 0xc000018030
fmt.Println("q 指向的值:", *q)         // q 指向的值: 25
```

`*类型` 就是指针类型。`*string` 是"字符串指针"，`*int` 是"整数指针"。

## 4. 用指针解决开头的问题

```go
// 参数类型改为 *User（User 的指针）
func birthday(u *User) {
    u.Age++
    fmt.Println("函数内部：", u.Age) // 26
}

func main() {
    user := User{Name: "张三", Age: 25}
    birthday(&user)  // 传入 user 的地址
    fmt.Println("函数外部：", user.Age) // 26 ← 变了！
}
```

过程解析：

```
调用前：
  main 的 user 在地址 0x100 → { Name:"张三", Age:25 }

调用时：
  函数的 u = 0x100（复制了一份地址值）
  u 和 &user 都指向 0x100 这同一块数据

函数内 u.Age++：
  地址 0x100 的数据变成 → { Name:"张三", Age:26 }
  main 的 user 还在 0x100 → 也看到了变化
```

**本质上仍然是值传递**——传递的是"地址这个值"的副本。但因为地址指向同一块内存，所以修改生效了。

## 5. 为什么 slice、map、channel "看起来像引用传递"？

### slice 的内部结构

切片不是简单的数据，它内部是一个"描述符"：

```
slice 变量 = {
    指向底层数组的指针,  ← 关键：这是一个指针
    长度 (len),
    容量 (cap)
}
```

传参时复制的是这个描述符，但**指针部分指向同一个底层数组**：

```go
func modifySlice(s []int) {
    s[0] = 999  // 通过指针修改了底层数组 → 外部能看到变化
    s = append(s, 100)  // 可能创建新数组，只改了局部变量 s
}

func main() {
    nums := []int{1, 2, 3}
    modifySlice(nums)
    fmt.Println(nums)  // [999 2 3]（s[0] 的修改生效了）
    // 但 append 的 100 不在里面，因为 append 可能创建了新底层数组
}
```

### map 的行为

map 本身就是一个指针（指向底层哈希表），所以函数内修改 key-value 会影响外部：

```go
func modifyMap(m map[string]int) {
    m["age"] = 30  // 影响外部
}

func main() {
    info := map[string]int{"age": 25}
    modifyMap(info)
    fmt.Println(info["age"])  // 30
}
```

但如果你**重新赋值整个 map 变量**，只影响局部：

```go
func replaceMap(m map[string]int) {
    m = map[string]int{"age": 99}  // 只改了局部变量 m
}

func main() {
    info := map[string]int{"age": 25}
    replaceMap(info)
    fmt.Println(info["age"])  // 25（没变）
}
```

### 总结对比

| 类型 | 传参行为 | 修改元素影响外部？ | 重新赋值影响外部？ |
|------|---------|-------------------|-------------------|
| int, string, bool | 复制值 | - | 不影响 |
| struct | 复制整个结构体 | 不影响 | 不影响 |
| *struct（指针） | 复制地址 | 影响 | 不影响 |
| slice | 复制描述符 | 影响 | 不影响 |
| map | 复制指针 | 影响 | 不影响 |
| channel | 复制指针 | 影响 | 不影响 |

## 6. 数组 vs 切片：值传递的直观对比

### 为什么数组和切片表现不同？

**数组是值类型**，切片是引用类型（内部含指针）。

```go
func modifyArray(a [3]int) {
    fmt.Println("函数内 - 修改前:", a)
    a[0] = 999
    fmt.Println("函数内 - 修改后:", a)
}

func modifySlice(s []int) {
    fmt.Println("函数内 - 修改前:", s)
    s[0] = 999
    fmt.Println("函数内 - 修改后:", s)
}

func main() {
    fmt.Println("===== 数组测试 =====")
    arr := [3]int{1, 2, 3}
    fmt.Println("调用前:", arr)
    modifyArray(arr)
    fmt.Println("调用后:", arr)  // [1 2 3]（没变！）
    
    fmt.Println("\n===== 切片测试 =====")
    sli := []int{1, 2, 3}
    fmt.Println("调用前:", sli)
    modifySlice(sli)
    fmt.Println("调用后:", sli)  // [999 2 3]（变了！）
}
```

**输出结果：**
```
===== 数组测试 =====
调用前: [1 2 3]
函数内 - 修改前: [1 2 3]
函数内 - 修改后: [999 2 3]
调用后: [1 2 3]

===== 切片测试 =====
调用前: [1 2 3]
函数内 - 修改前: [1 2 3]
函数内 - 修改后: [999 2 3]
调用后: [999 2 3]
```

### 内存图解

**数组传参（完整复制）：**

```
main 函数的内存：
  arr 变量 → [1, 2, 3]  ← 原始数组

调用 modifyArray(arr) 时：
  Go 复制整个数组
  
  main 的 arr     → [1, 2, 3]  ← 原始数组（独立副本）
  函数的 a 参数   → [1, 2, 3]  ← 复制的副本（独立副本）

函数内 a[0] = 999：
  main 的 arr     → [1, 2, 3]  ← 没变（两个独立的数组）
  函数的 a 参数   → [999, 2, 3]  ← 只有副本变了
```

**切片传参（共享底层数组）：**

```
main 函数的内存：
  底层数组（在堆上）→ [1, 2, 3]  ← 真实数据
  sli 变量 → {指针→底层数组, len:3, cap:3}

调用 modifySlice(sli) 时：
  Go 复制切片的"描述符"（但指针指向同一个底层数组）
  
  main 的 sli     → {指针→底层数组, len:3, cap:3}
  函数的 s 参数   → {指针→底层数组, len:3, cap:3}  ← 指针指向同一个数组！

函数内 s[0] = 999：
  底层数组 → [999, 2, 3]  ← 修改了真实数据
  main 的 sli 看到的也是 [999, 2, 3]  ← 因为指向同一个数组
```

### 关键区别总结

| 特性 | 数组 `[3]int` | 切片 `[]int` |
|------|--------------|-----------|
| 是否值类型 | ✅ 是 | ❌ 否（内部含指针） |
| 传参时 | 复制整个数组 | 复制描述符（指针指向同一数组） |
| 修改元素影响外部 | ❌ 不影响 | ✅ 影响 |
| 内存占用 | 固定（3个int） | 固定（指针+len+cap） |
| 大数组传参 | ⚠️ 低效（复制整个数组） | ✅ 高效（只复制描述符） |

### 实战建议

```go
// ❌ 不推荐：大数组作为参数
func processArray(arr [1000]int) {  // 复制 1000 个 int！
    // ...
}

// ✅ 推荐方案 1：用切片
func processSlice(s []int) {  // 只复制描述符
    // ...
}

// ✅ 推荐方案 2：用数组指针
func processArrayPtr(arr *[1000]int) {  // 只复制指针
    // ...
}
```

## 7. nil 指针：小心 panic

解引用一个 nil 指针会导致程序崩溃：

```go
var p *User = nil
fmt.Println(p.Name)  // panic: runtime error: invalid memory address
```

**安全做法**：使用前检查 nil：

```go
func printUser(u *User) {
    if u == nil {
        fmt.Println("用户不存在")
        return
    }
    fmt.Printf("%s, %d岁\n", u.Name, u.Age)
}
```

## 8. new 和 & 的区别

两种创建指针的方式：

```go
// 方式一：& 取地址（更常用）
u1 := &User{Name: "张三", Age: 25}

// 方式二：new（创建零值并返回指针）
u2 := new(User)  // u2 是 *User，所有字段是零值
u2.Name = "李四"
u2.Age = 30

// 效果等价于：
u3 := &User{}
```

`&User{...}` 更常用，因为可以同时初始化字段。

## 9. 什么时候该用指针？

| 场景 | 建议 |
|------|------|
| 函数需要修改参数 | 用指针 `*Type` |
| 结构体很大（字段很多） | 用指针（避免复制开销） |
| 方法需要修改接收者 | 用指针接收者 `func (t *Type)` |
| 表示"可能为空"的值 | 用指针（nil 表示不存在） |
| 小的基本类型（int, string 等） | 用值（复制成本低） |
| 只读不修改 | 用值（更安全） |

**实践经验**：在企业开发中，结构体参数和方法接收者几乎都用指针。

## 10. 综合实战：银行账户

```go
package main

import (
    "errors"
    "fmt"
)

type Account struct {
    Owner   string
    Balance float64
}

func NewAccount(owner string, initialBalance float64) *Account {
    return &Account{
        Owner:   owner,
        Balance: initialBalance,
    }
}

func (a *Account) Deposit(amount float64) error {
    if amount <= 0 {
        return errors.New("存款金额必须大于零")
    }
    a.Balance += amount
    fmt.Printf("[%s] 存入 %.2f，余额 %.2f\n", a.Owner, amount, a.Balance)
    return nil
}

func (a *Account) Withdraw(amount float64) error {
    if amount <= 0 {
        return errors.New("取款金额必须大于零")
    }
    if amount > a.Balance {
        return fmt.Errorf("余额不足（当前 %.2f，需要 %.2f）", a.Balance, amount)
    }
    a.Balance -= amount
    fmt.Printf("[%s] 取出 %.2f，余额 %.2f\n", a.Owner, amount, a.Balance)
    return nil
}

func Transfer(from, to *Account, amount float64) error {
    if err := from.Withdraw(amount); err != nil {
        return fmt.Errorf("转账失败: %w", err)
    }
    if err := to.Deposit(amount); err != nil {
        from.Deposit(amount) // 回滚
        return fmt.Errorf("转账失败: %w", err)
    }
    fmt.Printf("转账成功：%s → %s %.2f 元\n", from.Owner, to.Owner, amount)
    return nil
}

func main() {
    alice := NewAccount("Alice", 1000)
    bob := NewAccount("Bob", 500)

    fmt.Println("===== 初始状态 =====")
    fmt.Printf("Alice: %.2f, Bob: %.2f\n\n", alice.Balance, bob.Balance)

    alice.Deposit(500)
    bob.Withdraw(200)

    fmt.Println()
    Transfer(alice, bob, 300)

    fmt.Printf("\n===== 最终状态 =====\n")
    fmt.Printf("Alice: %.2f, Bob: %.2f\n", alice.Balance, bob.Balance)

    fmt.Println()
    err := Transfer(bob, alice, 9999)
    if err != nil {
        fmt.Println("错误：", err)
    }
}
```

**运行输出：**

```
===== 初始状态 =====
Alice: 1000.00, Bob: 500.00

[Alice] 存入 500.00，余额 1500.00
[Bob] 取出 200.00，余额 300.00

[Alice] 取出 300.00，余额 1200.00
[Bob] 存入 300.00，余额 600.00
转账成功：Alice → Bob 300.00 元

===== 最终状态 =====
Alice: 1200.00, Bob: 600.00

[Bob] 取出 9999.00，余额 -9699.00
错误：转账失败: 余额不足（当前 600.00，需要 9999.00）
```

### 代码要点解析

1. **指针接收者** `(a *Account)` - 方法能修改 Account 的字段
2. **指针参数** `Transfer(from, to *Account, ...)` - 函数能修改账户余额
3. **错误处理** - 转账失败时自动回滚（`from.Deposit(amount)`）
4. **实际效果** - Alice 最终余额 1200（1000+500-300），Bob 最终余额 600（500-200+300）

## 11. 本章小结

| 概念 | 说明 |
|------|------|
| 值传递 | Go 参数传递永远是复制值 |
| 指针 | 存放变量地址的变量 |
| `&` | 取地址运算符 |
| `*` | 解引用 / 指针类型声明 |
| slice/map/channel | 内部含指针，修改元素会影响外部 |
| nil 指针 | 解引用会 panic，使用前要检查 |
| 使用建议 | 需要修改或大结构体用指针，小值类型用值 |
