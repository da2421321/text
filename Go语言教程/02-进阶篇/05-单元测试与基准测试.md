## 1. 为什么要写测试

想象一下，你买了一部新手机。在手机出厂前，厂家会进行各种测试：屏幕是否正常显示、按键是否灵敏、摄像头能否拍照、电池能否充电……这些测试确保你拿到手的产品是合格的。

编写代码也是一样的道理：

- **质量保证**：测试能发现代码中的 bug，确保功能正常
- **重构信心**：有了测试，你可以放心地修改代码，测试会告诉你是否破坏了原有功能
- **文档作用**：测试代码展示了函数的使用方式和预期行为
- **设计改进**：难以测试的代码往往设计不够好，写测试能促使你改进设计

Go 语言内置了强大的测试框架，让测试变得简单而优雅。

## 2. Go 测试基础

### 2.1 testing 包

Go 标准库提供了 `testing` 包，无需安装第三方库就能编写测试。

### 2.2 文件命名规范

测试文件必须以 `_test.go` 结尾，例如：

```
calculator.go       // 源代码文件
calculator_test.go  // 测试文件
```

### 2.3 测试函数命名规范

测试函数必须：
- 以 `Test` 开头
- 接收一个 `*testing.T` 类型的参数
- 函数名格式：`TestXxx`（Xxx 通常是被测试的函数名）

```go
func TestAdd(t *testing.T) {
    // 测试代码
}
```

### 2.4 运行测试

```bash
go test              # 运行当前目录下的所有测试
go test -v           # 显示详细输出
go test ./...        # 运行当前目录及子目录的所有测试
go test -run TestAdd # 只运行名称匹配的测试
```

## 3. 编写第一个测试

### 3.1 创建被测试的代码

创建文件 `math.go`：

```go
package math

import "fmt"

// Add 计算两个整数的和
func Add(a, b int) int {
    return a + b
}

// Subtract 计算两个整数的差
func Subtract(a, b int) int {
    return a - b
}

// Multiply 计算两个整数的积
func Multiply(a, b int) int {
    return a * b
}

// Divide 计算两个整数的商，返回商和错误
func Divide(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("除数不能为零")
    }
    return a / b, nil
}
```

### 3.2 编写测试代码

创建文件 `math_test.go`：

```go
package math

import "testing"

func TestAdd(t *testing.T) {
    result := Add(2, 3)
    expected := 5

    if result != expected {
        t.Errorf("Add(2, 3) = %d; 期望 %d", result, expected)
    }
}

func TestSubtract(t *testing.T) {
    result := Subtract(5, 3)
    expected := 2

    if result != expected {
        t.Errorf("Subtract(5, 3) = %d; 期望 %d", result, expected)
    }
}

func TestDivide(t *testing.T) {
    // 测试正常情况
    result, err := Divide(10, 2)
    if err != nil {
        t.Fatalf("Divide(10, 2) 返回错误: %v", err)
    }
    if result != 5 {
        t.Errorf("Divide(10, 2) = %d; 期望 5", result)
    }

    // 测试除零情况
    _, err = Divide(10, 0)
    if err == nil {
        t.Error("Divide(10, 0) 应该返回错误，但没有")
    }
}
```

### 3.3 testing.T 的常用方法

- `t.Log(args...)` / `t.Logf(format, args...)`：记录日志（只在测试失败或 -v 时显示）
- `t.Error(args...)` / `t.Errorf(format, args...)`：标记测试失败，继续执行
- `t.Fatal(args...)` / `t.Fatalf(format, args...)`：标记测试失败，立即停止当前测试
- `t.Fail()`：标记测试失败，继续执行
- `t.FailNow()`：标记测试失败，立即停止

**何时使用 Error vs Fatal？**

- 使用 `Error`：当一个测试有多个检查点，即使前面失败也想看后面的结果
- 使用 `Fatal`：当后续测试依赖前面的结果，前面失败了后面没意义

## 4. 表驱动测试（Table-Driven Tests）

表驱动测试是 Go 社区最推荐的测试模式，它将测试数据和测试逻辑分离，让测试更清晰、更易维护。

### 4.1 什么是表驱动测试

就像餐厅的菜单一样，把所有测试用例列成一张表，然后逐个执行。

### 4.2 表驱动测试示例

```go
package math

import "testing"

func TestAddTable(t *testing.T) {
    // 定义测试用例表
    tests := []struct {
        name     string // 测试用例名称
        a        int    // 输入参数 a
        b        int    // 输入参数 b
        expected int    // 期望结果
    }{
        {"正数相加", 2, 3, 5},
        {"负数相加", -2, -3, -5},
        {"正负相加", 5, -3, 2},
        {"加零", 5, 0, 5},
        {"零加零", 0, 0, 0},
    }

    // 遍历测试用例
    for _, tt := range tests {
        result := Add(tt.a, tt.b)
        if result != tt.expected {
            t.Errorf("%s: Add(%d, %d) = %d; 期望 %d",
                tt.name, tt.a, tt.b, result, tt.expected)
        }
    }
}
```
