# 第一章：环境搭建与第一个程序

> 本章目标：在你的电脑上安装 Go 语言环境，写出并运行第一个 Go 程序。

## 1.1 什么是编程？

编程就是用一种计算机能理解的语言，告诉计算机要做什么。就像你用中文和人交流，你用 **Go 语言**和计算机交流。

你写的代码（一段文字）会被"翻译"成计算机能执行的指令，这个翻译过程叫做**编译**。Go 语言就是一种**编译型语言**。

```
你写的 Go 代码 → 编译器翻译 → 可执行文件 → 计算机运行
```

## 1.2 安装 Go

### Windows 安装

1. 打开浏览器，访问 [https://go.dev/dl/](https://go.dev/dl/)
2. 下载 Windows 版本（`go1.21.x.windows-amd64.msi`）
3. 双击下载的文件，一直点"下一步"即可完成安装
4. 默认安装路径是 `C:\Program Files\Go`

### macOS 安装

1. 访问 [https://go.dev/dl/](https://go.dev/dl/)
2. 下载 macOS 版本（`go1.21.x.darwin-amd64.pkg` 或 `arm64`）
3. 双击安装包，按提示完成安装

或者用 Homebrew：
```bash
brew install go
```

### Linux 安装

```bash
# 下载（以 1.21 为例，请去官网确认最新版本号）
wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz

# 解压到 /usr/local
sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz

# 添加到环境变量（写入 ~/.bashrc 或 ~/.zshrc）
export PATH=$PATH:/usr/local/go/bin
```

### 验证安装成功

安装完成后，打开**终端**（Windows 用户打开 PowerShell 或 CMD），输入：

```bash
go version
```

如果看到类似以下输出，说明安装成功：
```
go version go1.21.0 windows/amd64
```

再试一下：
```bash
go env GOPATH
```

这会显示 Go 的工作目录路径，不需要修改，知道它存在就行。

## 1.3 安装代码编辑器

推荐使用 **VS Code**（Visual Studio Code），它免费、轻量、插件丰富。

1. 下载安装 VS Code：[https://code.visualstudio.com/](https://code.visualstudio.com/)
2. 打开 VS Code，点击左侧的"扩展"图标（四个方块的图标）
3. 搜索 `Go`，安装由 Go Team at Google 发布的官方插件
4. 安装完成后，VS Code 会自动提示安装 Go 工具包，点击 **Install All**

安装 Go 插件后，你将获得：
- **语法高亮**：代码会用不同颜色显示，方便阅读
- **代码补全**：输入部分代码后会自动提示
- **自动格式化**：保存文件时自动整理代码格式
- **错误提示**：写错了会在代码下面画红线

## 1.4 创建第一个项目

### 第一步：创建项目文件夹

在你喜欢的位置创建一个文件夹，比如在桌面上创建 `learn-go`：

```bash
# Windows (PowerShell)
mkdir ~/Desktop/learn-go
cd ~/Desktop/learn-go

# macOS / Linux
mkdir ~/Desktop/learn-go
cd ~/Desktop/learn-go
```

### 第二步：初始化 Go 项目

在终端中输入：

```bash
go mod init learn-go
```

这条命令会创建一个 `go.mod` 文件，它的作用类似于一个"项目说明书"，告诉 Go：
- 这个项目叫什么名字
- 使用哪个版本的 Go
- 依赖了哪些外部库（目前没有）

执行后会看到文件夹里多了一个 `go.mod` 文件，内容类似：
```
module learn-go

go 1.21
```

### 第三步：创建第一个 Go 文件

用 VS Code 打开 `learn-go` 文件夹，然后新建一个文件，命名为 `main.go`。

在文件中输入以下代码：

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

**先别急，我们逐行解释这段代码：**

### 逐行解读

```go
package main
```
每个 Go 文件的第一行必须声明它属于哪个**包（package）**。`main` 是一个特殊的包名，表示"这是一个可以运行的程序"。

```go
import "fmt"
```
`import` 的意思是"导入"，就是引入别人写好的代码库。`fmt`（format 的缩写）是 Go 自带的格式化输出库，提供了打印文字到屏幕上的功能。

```go
func main() {
```
`func` 是 function（函数）的缩写，表示"定义一个函数"。`main` 是函数的名字。`main` 包中的 `main` 函数是整个程序的**入口点**——程序从这里开始执行。

花括号 `{` 表示函数体的开始。注意：Go 要求**左花括号必须和 `func` 在同一行**，不能另起一行。

```go
    fmt.Println("Hello, World!")
```
`fmt.Println` 的意思是：使用 `fmt` 包中的 `Println` 函数（Print Line，打印一行）。括号里的 `"Hello, World!"` 是要打印的内容，用双引号包裹的文字叫做**字符串**。

```go
}
```
右花括号表示函数体结束。

## 1.5 运行程序

### 方法一：go run（快速运行）

在终端中，确保你在项目目录下，输入：

```bash
go run main.go
```

你会看到屏幕上输出：
```
Hello, World!
```

`go run` 的意思是"编译并立即运行"，它不会留下可执行文件。适合开发和测试时使用。

### 方法二：go build（编译成可执行文件）

```bash
go build -o myapp main.go
```

这会在当前目录生成一个名为 `myapp` 的可执行文件（Windows 上是 `myapp.exe`）。然后你可以直接运行它：

```bash
# Windows
.\myapp.exe

# macOS / Linux
./myapp
```

输出同样是：
```
Hello, World!
```

`go build` 适合发布和部署时使用。Go 的一大优势就是编译后只有一个文件，拷贝到任何同系统的电脑上就能运行，不需要安装 Go 环境。

## 1.6 多试几个例子

### 打印多行文字

```go
package main

import "fmt"

func main() {
    fmt.Println("你好，世界！")
    fmt.Println("我正在学习 Go 语言")
    fmt.Println("Go 语言很有趣！")
}
```

输出：
```
你好，世界！
我正在学习 Go 语言
Go 语言很有趣！
```

### 做简单的数学计算

```go
package main

import "fmt"

func main() {
    fmt.Println("1 + 2 =", 1+2)
    fmt.Println("10 - 3 =", 10-3)
    fmt.Println("4 * 5 =", 4*5)
    fmt.Println("10 / 3 =", 10/3)   // 整数除法，结果是 3
    fmt.Println("10 % 3 =", 10%3)   // 取余数，结果是 1
}
```

输出：
```
1 + 2 = 3
10 - 3 = 7
4 * 5 = 20
10 / 3 = 3
10 % 3 = 1
```

> 注意：`10 / 3` 的结果是 `3` 而不是 `3.333`，因为两个整数相除，结果也是整数。后面学数据类型时会详细解释。

## 1.7 常见问题排查

### 问题 1：运行时提示 "go: command not found"

说明 Go 没有正确安装，或者没有添加到系统的 PATH 环境变量中。
- Windows：重新运行安装程序，确保勾选了"Add to PATH"
- macOS/Linux：检查 `~/.bashrc` 或 `~/.zshrc` 中是否有 Go 的路径配置

### 问题 2：左花括号另起一行报错

```go
// 错误写法！Go 不允许这样写
func main()
{
    fmt.Println("Hello")
}
```

Go 有严格的代码风格要求：**左花括号必须和前面的语句在同一行**。

```go
// 正确写法
func main() {
    fmt.Println("Hello")
}
```

### 问题 3：导入的包没有使用

```go
import "fmt"
import "os"  // 导入了但没用到

func main() {
    fmt.Println("Hello")
}
```

Go 编译器会直接报错：`imported and not used: "os"`。在 Go 中，导入了就必须使用，不允许有"废物"代码。这是 Go 的设计哲学：保持代码整洁。

### 问题 4：提示 "cannot find package"

确保你已经执行了 `go mod init`。没有 `go.mod` 文件的目录，Go 不知道该怎么管理依赖。

## 1.8 常用 Go 命令速查

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `go run main.go` | 编译并运行 | 开发测试 |
| `go build` | 编译成可执行文件 | 发布部署 |
| `go mod init 项目名` | 初始化项目 | 创建新项目 |
| `go mod tidy` | 整理依赖 | 添加/删除库后 |
| `go fmt ./...` | 格式化代码 | 统一代码风格 |
| `go vet ./...` | 检查代码问题 | 提交前检查 |
| `go test ./...` | 运行测试 | 测试代码 |

## 1.9 本章小结

- Go 是一门编译型语言，代码需要先编译再运行
- 每个 Go 程序都从 `package main` 的 `func main()` 开始执行
- `import` 用于导入需要的包，`fmt.Println` 用于打印输出
- `go run` 快速运行，`go build` 编译成可执行文件
- Go 对代码风格有严格要求：花括号不能另起一行，导入的包必须使用

## 1.10 练习题

1. **修改 Hello World**：把输出的文字改成你自己的名字，比如 `"你好，我是张三！"`
2. **多行输出**：用 3 个 `fmt.Println` 打印出你的姓名、年龄和城市
3. **数学计算**：用 `fmt.Println` 计算并输出 `123 + 456` 和 `100 * 99` 的结果
4. **编译运行**：用 `go build` 把程序编译成可执行文件，然后运行它
