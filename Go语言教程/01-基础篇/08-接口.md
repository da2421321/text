# 第八章：接口

> 本章目标：理解 Go 接口的核心思想——"隐式实现"，掌握接口的定义、实现和常见用法。

## 8.1 什么是接口？

想象一个场景：你有一台打印机，它不关心你给它的是 Word 文档、PDF 还是图片，只要你给它的东西"能被打印"就行。

**接口（interface）** 就是这种思想：它定义了一组**行为**（方法），任何类型只要实现了这些方法，就满足了这个接口。

```
接口 = "你能做什么"
结构体 = "你是什么 + 你怎么做"
```

## 8.2 定义接口

```go
type 接口名 interface {
    方法名(参数) 返回值
    方法名(参数) 返回值
    ...
}
```

实际例子：

```go
// 定义一个 Animal 接口，要求实现 Speak 方法
type Animal interface {
    Speak() string
}
```

这个接口说的是："任何能 `Speak()` 的东西，都算是 `Animal`"。

## 8.3 实现接口（隐式实现）

Go 的接口实现方式非常特别：**不需要显式声明 `implements`**。只要一个类型实现了接口要求的所有方法，它就自动实现了该接口。

```go
type Animal interface {
    Speak() string
}

// Dog 结构体
type Dog struct {
    Name string
}

// Dog 实现了 Speak() 方法 → 自动实现了 Animal 接口
func (d Dog) Speak() string {
    return d.Name + " 说：汪汪！"
}

// Cat 结构体
type Cat struct {
    Name string
}

// Cat 也实现了 Speak() 方法 → 也自动实现了 Animal 接口
func (c Cat) Speak() string {
    return c.Name + " 说：喵喵！"
}

func main() {
    // Dog 和 Cat 都可以赋值给 Animal 类型的变量
    var a1 Animal = Dog{Name: "旺财"}
    var a2 Animal = Cat{Name: "咪咪"}

    fmt.Println(a1.Speak()) // 旺财 说：汪汪！
    fmt.Println(a2.Speak()) // 咪咪 说：喵喵！
}
```

和 Java 的区别：
- Java：`class Dog implements Animal { ... }` — 必须显式声明
- Go：只要方法签名匹配就自动实现 — 不需要任何声明

## 8.4 接口的真正价值：多态

**多态**是指：同一个函数，传入不同类型的参数，执行不同的行为。

```go
// 这个函数接受任何 Animal
func MakeSound(a Animal) {
    fmt.Println(a.Speak())
}

func main() {
    dog := Dog{Name: "旺财"}
    cat := Cat{Name: "咪咪"}

    MakeSound(dog) // 旺财 说：汪汪！
    MakeSound(cat) // 咪咪 说：喵喵！

    // 存入切片，统一处理
    animals := []Animal{
        Dog{Name: "大黄"},
        Cat{Name: "小白"},
        Dog{Name: "阿旺"},
    }

    for _, a := range animals {
        MakeSound(a)
    }
}
```

这就是接口的威力：`MakeSound` 函数不关心传入的具体是狗还是猫，只要它能 `Speak()` 就行。

## 8.5 更实际的例子：支付系统

```go
// 定义支付接口
type PaymentMethod interface {
    Pay(amount float64) string
    Name() string
}

// 支付宝
type Alipay struct {
    Account string
}

func (a Alipay) Pay(amount float64) string {
    return fmt.Sprintf("通过支付宝（%s）支付了 %.2f 元", a.Account, amount)
}

func (a Alipay) Name() string {
    return "支付宝"
}

// 微信支付
type WechatPay struct {
    OpenID string
}

func (w WechatPay) Pay(amount float64) string {
    return fmt.Sprintf("通过微信支付（%s）支付了 %.2f 元", w.OpenID, amount)
}

func (w WechatPay) Name() string {
    return "微信支付"
}

// 银行卡
type BankCard struct {
    CardNumber string
    BankName   string
}

func (b BankCard) Pay(amount float64) string {
    return fmt.Sprintf("通过%s（尾号%s）支付了 %.2f 元",
        b.BankName, b.CardNumber[len(b.CardNumber)-4:], amount)
}

func (b BankCard) Name() string {
    return b.BankName
}

// 统一的结账函数——不关心用什么方式支付
func Checkout(method PaymentMethod, amount float64) {
    fmt.Printf("使用 %s 付款...\n", method.Name())
    result := method.Pay(amount)
    fmt.Println(result)
    fmt.Println("支付完成！\n")
}

func main() {
    alipay := Alipay{Account: "zhangsan@alipay.com"}
    wechat := WechatPay{OpenID: "wx_abc123"}
    bank := BankCard{CardNumber: "6222021234567890", BankName: "工商银行"}

    Checkout(alipay, 99.9)
    Checkout(wechat, 199.0)
    Checkout(bank, 5000.0)
}
```

这个例子展示了接口的实际价值：
- 新增支付方式（如 Apple Pay）时，只需新增一个结构体和它的方法
- `Checkout` 函数完全不需要修改
- 这就是**"对扩展开放，对修改关闭"**的设计原则

## 8.6 空接口 `any`

空接口 `interface{}` 不包含任何方法，因此**任何类型都实现了空接口**。

Go 1.18+ 引入了 `any` 作为 `interface{}` 的别名：

```go
// 可以接受任意类型
func PrintAnything(v any) {
    fmt.Println(v)
}

func main() {
    PrintAnything(42)           // 42
    PrintAnything("hello")      // hello
    PrintAnything(true)         // true
    PrintAnything([]int{1,2,3}) // [1 2 3]
}
```

### 类型断言

当你拿到一个 `any` 类型的值，想知道它具体是什么类型时，用**类型断言**：

```go
func describe(v any) {
    // 安全的类型断言（推荐）
    if s, ok := v.(string); ok {
        fmt.Println("这是字符串：", s)
        return
    }
    if n, ok := v.(int); ok {
        fmt.Println("这是整数：", n)
        return
    }
    fmt.Println("未知类型")
}

func main() {
    describe("hello") // 这是字符串：hello
    describe(42)       // 这是整数：42
    describe(3.14)     // 未知类型
}
```

> **注意**：不安全的断言 `v.(string)` 如果类型不匹配会 panic。始终使用双返回值模式 `s, ok := v.(string)`。

### type switch

处理多种类型时，`type switch` 更优雅：

```go
func describe(v any) {
    switch x := v.(type) {
    case string:
        fmt.Printf("字符串：%s（长度 %d）\n", x, len(x))
    case int:
        fmt.Printf("整数：%d\n", x)
    case float64:
        fmt.Printf("浮点数：%.2f\n", x)
    case bool:
        fmt.Printf("布尔值：%t\n", x)
    case []int:
        fmt.Printf("整数切片：%v（长度 %d）\n", x, len(x))
    default:
        fmt.Printf("未知类型：%T\n", x)
    }
}
```

## 8.7 接口组合

小接口可以组合成大接口：

```go
type Reader interface {
    Read(data []byte) (int, error)
}

type Writer interface {
    Write(data []byte) (int, error)
}

// 组合：ReadWriter 同时要求 Read 和 Write
type ReadWriter interface {
    Reader
    Writer
}
```

Go 标准库大量使用这种模式。比如 `io.ReadWriter` 就是由 `io.Reader` 和 `io.Writer` 组合而成。

**设计建议**：接口要尽量小。一个接口一个方法是最理想的。Go 有句名言：

> "The bigger the interface, the weaker the abstraction."
> 接口越大，抽象能力越弱。

## 8.8 接口的 nil 陷阱（重要）

这是面试高频考点，也是实际开发中容易犯的错误：

```go
type Animal interface {
    Speak() string
}

type Dog struct{}

func (d *Dog) Speak() string { return "汪汪" }

func main() {
    var d *Dog = nil
    var a Animal = d

    fmt.Println(d == nil) // true
    fmt.Println(a == nil) // false ← 注意！
}
```

为什么 `a` 不是 nil？因为接口值由两部分组成：

```
接口值 = (动态类型, 动态值)

d = nil              → d 是 nil
a = (*Dog, nil)      → a 的类型是 *Dog，值是 nil
                       类型不为空，所以 a != nil
```

**安全做法**：

```go
// 不要把具体类型的 nil 赋给接口
// 应该直接赋 nil 给接口变量
var a Animal = nil  // 正确：(nil, nil)
```

## 8.9 常见标准库接口

Go 标准库中有很多常用接口，了解它们能帮你写出更地道的 Go 代码：

| 接口 | 方法 | 用途 |
|------|------|------|
| `fmt.Stringer` | `String() string` | 自定义打印格式 |
| `error` | `Error() string` | 错误处理 |
| `io.Reader` | `Read(p []byte) (n int, err error)` | 读取数据 |
| `io.Writer` | `Write(p []byte) (n int, err error)` | 写入数据 |
| `sort.Interface` | `Len()`, `Less()`, `Swap()` | 自定义排序 |

### Stringer 接口示例

```go
type User struct {
    Name string
    Age  int
}

// 实现 fmt.Stringer 接口
func (u User) String() string {
    return fmt.Sprintf("%s（%d岁）", u.Name, u.Age)
}

func main() {
    u := User{Name: "张三", Age: 25}
    fmt.Println(u) // 张三（25岁）← 自动调用 String() 方法
}
```

## 8.10 综合实战：图形计算器

```go
package main

import (
    "fmt"
    "math"
)

type Shape interface {
    Area() float64
    Perimeter() float64
    Name() string
}

type Circle struct {
    Radius float64
}

func (c Circle) Area() float64      { return math.Pi * c.Radius * c.Radius }
func (c Circle) Perimeter() float64 { return 2 * math.Pi * c.Radius }
func (c Circle) Name() string       { return "圆形" }

type Rectangle struct {
    Width, Height float64
}

func (r Rectangle) Area() float64      { return r.Width * r.Height }
func (r Rectangle) Perimeter() float64 { return 2 * (r.Width + r.Height) }
func (r Rectangle) Name() string       { return "矩形" }

type Triangle struct {
    A, B, C float64 // 三条边
}

func (t Triangle) Perimeter() float64 { return t.A + t.B + t.C }
func (t Triangle) Area() float64 {
    s := t.Perimeter() / 2
    return math.Sqrt(s * (s - t.A) * (s - t.B) * (s - t.C))
}
func (t Triangle) Name() string { return "三角形" }

func PrintShapeInfo(s Shape) {
    fmt.Printf("【%s】面积 = %.2f，周长 = %.2f\n", s.Name(), s.Area(), s.Perimeter())
}

func TotalArea(shapes []Shape) float64 {
    total := 0.0
    for _, s := range shapes {
        total += s.Area()
    }
    return total
}

func main() {
    shapes := []Shape{
        Circle{Radius: 5},
        Rectangle{Width: 10, Height: 4},
        Triangle{A: 3, B: 4, C: 5},
    }

    for _, s := range shapes {
        PrintShapeInfo(s)
    }

    fmt.Printf("\n所有图形的总面积：%.2f\n", TotalArea(shapes))
}
```

输出：
```
【圆形】面积 = 78.54，周长 = 31.42
【矩形】面积 = 40.00，周长 = 28.00
【三角形】面积 = 6.00，周长 = 12.00

所有图形的总面积：124.54
```

## 8.11 本章小结

| 概念 | 说明 |
|------|------|
| 接口定义 | `type Name interface { Method() }` |
| 隐式实现 | 不需要 `implements`，方法匹配即实现 |
| 多态 | 不同类型可以传给同一个接口参数 |
| 空接口 `any` | 可以接受任意类型 |
| 类型断言 | `v.(string)` / `v.(type)` — 判断具体类型 |
| 接口组合 | 小接口组合成大接口 |
| nil 陷阱 | 接口值 = (类型, 值)，类型不为空则接口不为 nil |
| 设计原则 | 接口越小越好，一个方法最理想 |
