# 第十章：包与模块管理

> 本章目标：理解 Go 的包（package）系统和模块（module）管理，学会组织项目代码和使用第三方库。

## 10.1 什么是包（Package）？

随着程序越来越大，不可能把所有代码写在一个文件里。**包**就是 Go 组织代码的方式——把相关的代码放在一个文件夹里，形成一个"包"。

你其实已经在用包了：

```go
import "fmt"      // fmt 是 Go 标准库的一个包
import "strings"  // strings 也是一个包
```

## 10.2 创建自己的包

### 项目结构

```
learn-go/
├── go.mod
├── main.go          ← 主程序（package main）
├── mathutil/
│   └── calc.go      ← 数学工具包（package mathutil）
└── stringutil/
    └── format.go    ← 字符串工具包（package stringutil）
```

### 步骤 1：创建 mathutil 包

`mathutil/calc.go`：

```go
package mathutil  // 包名和文件夹名保持一致

func Add(a, b int) int {
    return a + b
}

func Max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

// 小写开头的函数是私有的，只能在 mathutil 包内使用
func helper() int {
    return 42
}
```

### 步骤 2：创建 stringutil 包

`stringutil/format.go`：

```go
package stringutil

import "strings"

func Capitalize(s string) string {
    if len(s) == 0 {
        return s
    }
    return strings.ToUpper(s[:1]) + s[1:]
}

func Repeat(s string, n int) string {
    return strings.Repeat(s, n)
}
```

### 步骤 3：在 main.go 中使用

```go
package main

import (
    "fmt"
    "learn-go/mathutil"      // 导入自定义包
    "learn-go/stringutil"    // 模块名/包名
)

func main() {
    sum := mathutil.Add(3, 5)
    fmt.Println("3 + 5 =", sum)  // 3 + 5 = 8

    max := mathutil.Max(10, 20)
    fmt.Println("较大的数是：", max)  // 较大的数是：20

    name := stringutil.Capitalize("hello")
    fmt.Println(name)  // Hello
}
```

导入路径格式：`模块名/包文件夹名`。模块名就是 `go.mod` 文件中 `module` 后面的名字。

## 10.3 包的核心规则

### 规则 1：每个文件夹一个包

同一个文件夹下的所有 `.go` 文件必须属于同一个包：

```
mathutil/
├── calc.go      ← package mathutil
├── geometry.go  ← package mathutil（同一个包）
└── stats.go     ← package mathutil（同一个包）
```

一个包可以有多个文件，它们共享包级别的变量和函数。

### 规则 2：大写字母开头 = 公开

这是 Go 最独特的设计之一：

```go
package mathutil

func Add(a, b int) int { ... }    // 大写开头 → 公开，外部可用
func helper() int { ... }         // 小写开头 → 私有，外部不可用

var MaxValue = 100                 // 大写开头 → 公开
var internalCount = 0              // 小写开头 → 私有

type Calculator struct { ... }     // 大写开头 → 公开
type cache struct { ... }          // 小写开头 → 私有
```

这个规则适用于：函数、变量、常量、类型、结构体字段、方法——一切！

### 规则 3：包名简短、小写、不用下划线

```go
// 好的包名
package http
package json
package mathutil

// 差的包名
package myMathUtilities    // 太长、有大写
package string_utils       // 有下划线
package util               // 太笼统
```

### 规则 4：导入的包必须使用

```go
import (
    "fmt"
    "os"  // 导入了但没用 → 编译报错
)
```

## 10.4 导入的几种方式

```go
// 标准导入
import "fmt"

// 批量导入（推荐）
import (
    "fmt"
    "strings"
    "learn-go/mathutil"
)

// 别名导入（当包名冲突时）
import (
    myMath "learn-go/mathutil"
    "math"
)
// 使用时：myMath.Add(1, 2)

// 点导入（不推荐，可能导致命名冲突）
import . "fmt"
// 使用时可以省略包名：Println("hello") 而不是 fmt.Println("hello")

// 匿名导入（只执行包的 init 函数，不直接使用包）
import _ "github.com/go-sql-driver/mysql"
```

## 10.5 go.mod 与模块管理

### 什么是模块（Module）？

**模块**是一组相关的包的集合，有一个统一的版本号。一个项目就是一个模块。

`go.mod` 文件定义了模块信息：

```
module learn-go

go 1.21

require (
    github.com/gin-gonic/gin v1.9.1
    gorm.io/gorm v1.25.5
)
```

### 初始化模块

```bash
# 创建新项目
mkdir my-project
cd my-project
go mod init my-project    # 或用域名：go mod init github.com/你的用户名/my-project
```

### 使用第三方库

以安装一个颜色输出库为例：

```bash
# 安装第三方库
go get github.com/fatih/color
```

使用：

```go
package main

import "github.com/fatih/color"

func main() {
    color.Red("这是红色文字")
    color.Green("这是绿色文字")
    color.Blue("这是蓝色文字")
}
```

安装后，`go.mod` 会自动更新：

```
module my-project

go 1.21

require github.com/fatih/color v1.16.0
```

同时会生成 `go.sum` 文件，记录每个依赖的精确版本和校验和（类似 `package-lock.json`）。

### 常用 go mod 命令

| 命令 | 作用 | 说明 |
|------|------|------|
| `go mod init 模块名` | 初始化模块 | 创建 go.mod |
| `go mod tidy` | 整理依赖 | 添加缺少的、删除多余的依赖 |
| `go get 包路径` | 安装/更新依赖 | 下载并记录到 go.mod |
| `go get 包路径@版本` | 安装指定版本 | 如 `@v1.2.3`、`@latest` |
| `go mod download` | 下载所有依赖 | 不修改 go.mod |
| `go mod vendor` | 复制依赖到 vendor 目录 | 离线使用 |
| `go list -m all` | 列出所有依赖 | 包括间接依赖 |

### go mod tidy（最常用）

每次修改代码后，运行 `go mod tidy` 来自动整理依赖：

```bash
go mod tidy
```

它会：
- 添加代码中 import 了但 go.mod 中没有的依赖
- 删除 go.mod 中有但代码中没用到的依赖

## 10.6 项目结构最佳实践

### 小项目

```
my-app/
├── go.mod
├── go.sum
├── main.go           ← 程序入口
├── handler.go        ← 处理逻辑
└── util.go           ← 工具函数
```

### 中型项目

```
my-app/
├── go.mod
├── go.sum
├── main.go
├── config/
│   └── config.go     ← 配置相关
├── handler/
│   ├── user.go       ← 用户处理
│   └── product.go    ← 商品处理
├── model/
│   ├── user.go       ← 用户模型
│   └── product.go    ← 商品模型
└── util/
    └── helper.go     ← 工具函数
```

### 大型项目（标准布局）

```
my-app/
├── cmd/
│   └── server/
│       └── main.go       ← 程序入口
├── internal/             ← 内部代码（不允许被外部导入）
│   ├── config/
│   ├── handler/
│   ├── service/
│   ├── repository/
│   └── model/
├── pkg/                  ← 可被外部导入的公共库
│   └── response/
├── go.mod
├── go.sum
├── Makefile
└── README.md
```

> `internal/` 是 Go 的特殊目录——里面的包只能被同模块的代码导入，外部项目无法使用。

## 10.7 init 函数

每个包可以有 `init` 函数，在包被导入时自动执行：

```go
package config

import "fmt"

var AppName string

func init() {
    AppName = "MyApp"
    fmt.Println("config 包已初始化")
}
```

执行顺序：
1. 被导入包的全局变量初始化
2. 被导入包的 `init()` 函数
3. 主包的全局变量初始化
4. 主包的 `init()` 函数
5. `main()` 函数

```
导入的包 init() → main 包 init() → main()
```

## 10.8 综合实战：组织一个计算器项目

```
calculator/
├── go.mod
├── main.go
├── calc/
│   ├── basic.go      ← 基本运算
│   └── advanced.go   ← 高级运算
└── display/
    └── format.go     ← 显示格式化
```

`calc/basic.go`：

```go
package calc

import "errors"

var ErrDivideByZero = errors.New("除数不能为零")

func Add(a, b float64) float64 { return a + b }
func Sub(a, b float64) float64 { return a - b }
func Mul(a, b float64) float64 { return a * b }

func Div(a, b float64) (float64, error) {
    if b == 0 {
        return 0, ErrDivideByZero
    }
    return a / b, nil
}
```

`calc/advanced.go`：

```go
package calc

import "math"

func Power(base, exp float64) float64 {
    return math.Pow(base, exp)
}

func Sqrt(x float64) (float64, error) {
    if x < 0 {
        return 0, errors.New("不能对负数开方")
    }
    return math.Sqrt(x), nil
}
```

`display/format.go`：

```go
package display

import "fmt"

func PrintResult(expression string, result float64) {
    fmt.Printf("  %s = %.4f\n", expression, result)
}

func PrintError(expression string, err error) {
    fmt.Printf("  %s → 错误: %s\n", expression, err)
}

func PrintHeader(title string) {
    fmt.Printf("\n===== %s =====\n", title)
}
```

`main.go`：

```go
package main

import (
    "calculator/calc"
    "calculator/display"
)

func main() {
    display.PrintHeader("基本运算")
    display.PrintResult("3 + 5", calc.Add(3, 5))
    display.PrintResult("10 - 4", calc.Sub(10, 4))
    display.PrintResult("6 * 7", calc.Mul(6, 7))

    result, err := calc.Div(10, 3)
    if err != nil {
        display.PrintError("10 / 3", err)
    } else {
        display.PrintResult("10 / 3", result)
    }

    _, err = calc.Div(10, 0)
    if err != nil {
        display.PrintError("10 / 0", err)
    }

    display.PrintHeader("高级运算")
    display.PrintResult("2^10", calc.Power(2, 10))

    sqrtResult, err := calc.Sqrt(144)
    if err != nil {
        display.PrintError("√144", err)
    } else {
        display.PrintResult("√144", sqrtResult)
    }
}
```

输出：
```
===== 基本运算 =====
  3 + 5 = 8.0000
  10 - 4 = 6.0000
  6 * 7 = 42.0000
  10 / 3 = 3.3333
  10 / 0 → 错误: 除数不能为零

===== 高级运算 =====
  2^10 = 1024.0000
  √144 = 12.0000
```

## 10.9 本章小结

| 概念 | 说明 |
|------|------|
| 包（package） | 代码组织单元，一个文件夹一个包 |
| 公开/私有 | 大写开头=公开，小写开头=私有 |
| import | 导入标准库或自定义包 |
| 模块（module） | 一组包的集合，由 go.mod 管理 |
| `go mod init` | 初始化模块 |
| `go mod tidy` | 整理依赖（最常用） |
| `go get` | 安装第三方库 |
| init 函数 | 包被导入时自动执行 |
| internal 目录 | 只能被同模块代码导入 |
