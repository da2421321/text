# 第九章：错误处理

> 本章目标：掌握 Go 独特的错误处理机制——`error` 接口、自定义错误、`panic` 和 `recover`。

## 9.1 Go 的错误处理哲学

大多数编程语言用 try/catch 捕获异常：

```
// Java / JavaScript / Python 的风格
try {
    result = riskyOperation()
} catch (error) {
    handleError(error)
}
```

Go 选择了一条不同的路：**把错误当作普通的返回值**，要求你显式处理每一个可能的错误。

```go
// Go 的风格
result, err := riskyOperation()
if err != nil {
    // 处理错误
}
// 继续正常流程
```

这种方式的好处：
- 错误处理就在发生错误的地方，不会被"隐藏"
- 你不会漏掉任何一个错误
- 代码的执行流程一目了然

## 9.2 error 接口

Go 中的错误类型是一个非常简单的接口：

```go
type error interface {
    Error() string
}
```

只有一个方法 `Error()`，返回错误描述字符串。**任何实现了这个方法的类型都可以作为 error**。

### 创建简单的错误

```go
import "errors"

// 方式一：errors.New（最常用）
err := errors.New("something went wrong")
fmt.Println(err) // something went wrong

// 方式二：fmt.Errorf（可以格式化）
name := "config.yaml"
err2 := fmt.Errorf("文件 %s 不存在", name)
fmt.Println(err2) // 文件 config.yaml 不存在
```

## 9.3 错误处理的基本模式

### 返回错误

函数通常把 `error` 作为最后一个返回值：

```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("除数不能为零")
    }
    return a / b, nil  // nil 表示"没有错误"
}
```

### 检查错误

调用者必须检查返回的 error：

```go
func main() {
    // 正常情况
    result, err := divide(10, 3)
    if err != nil {
        fmt.Println("出错了：", err)
        return
    }
    fmt.Printf("结果：%.2f\n", result) // 结果：3.33

    // 错误情况
    result2, err2 := divide(10, 0)
    if err2 != nil {
        fmt.Println("出错了：", err2) // 出错了：除数不能为零
        return
    }
    fmt.Println(result2) // 不会执行到这里
}
```

### if err != nil 模式

这是 Go 中最常见的代码模式：

```go
file, err := os.Open("data.txt")
if err != nil {
    fmt.Println("打开文件失败：", err)
    return
}
defer file.Close()

data, err := io.ReadAll(file)
if err != nil {
    fmt.Println("读取文件失败：", err)
    return
}

fmt.Println(string(data))
```

是的，你会看到大量的 `if err != nil`。这就是 Go 的风格，虽然看起来啰嗦，但保证了每个错误都被处理。

## 9.4 自定义错误类型

当简单的错误信息不够用时，可以创建自定义错误类型：

```go
// 自定义错误类型
type ValidationError struct {
    Field   string
    Message string
}

// 实现 error 接口
func (e *ValidationError) Error() string {
    return fmt.Sprintf("字段 '%s' 验证失败：%s", e.Field, e.Message)
}

func validateAge(age int) error {
    if age < 0 {
        return &ValidationError{
            Field:   "age",
            Message: "年龄不能为负数",
        }
    }
    if age > 150 {
        return &ValidationError{
            Field:   "age",
            Message: "年龄不能超过150",
        }
    }
    return nil
}

func main() {
    err := validateAge(-5)
    if err != nil {
        fmt.Println(err) // 字段 'age' 验证失败：年龄不能为负数

        // 可以类型断言获取更多信息
        if ve, ok := err.(*ValidationError); ok {
            fmt.Println("出错的字段：", ve.Field)
            fmt.Println("错误信息：", ve.Message)
        }
    }
}
```

## 9.5 错误包装与解包（Go 1.13+）

### 包装错误：添加上下文

当一个函数调用另一个函数并返回错误时，通常需要添加上下文信息：

```go
func readConfig(filename string) (string, error) {
    data, err := os.ReadFile(filename)
    if err != nil {
        // 用 %w 包装原始错误，保留错误链
        return "", fmt.Errorf("读取配置文件失败: %w", err)
    }
    return string(data), nil
}
```

### 解包错误：判断错误类型

```go
import "errors"

func main() {
    _, err := readConfig("config.yaml")
    if err != nil {
        fmt.Println(err)
        // 输出：读取配置文件失败: open config.yaml: no such file or directory

        // errors.Is：判断错误链中是否包含特定错误
        if errors.Is(err, os.ErrNotExist) {
            fmt.Println("文件不存在，请创建配置文件")
        }

        // errors.As：判断错误链中是否包含特定错误类型
        var pathErr *os.PathError
        if errors.As(err, &pathErr) {
            fmt.Println("路径错误，路径是：", pathErr.Path)
        }
    }
}
```

`%w` 和 `errors.Is` / `errors.As` 配合使用，能在保留错误上下文的同时，让调用者判断具体的错误原因。

## 9.6 哨兵错误（Sentinel Errors）

预定义一些常见的错误常量，方便调用者判断：

```go
var (
    ErrNotFound     = errors.New("资源不存在")
    ErrUnauthorized = errors.New("未授权")
    ErrForbidden    = errors.New("无权访问")
)

func findUser(id int) (*User, error) {
    if id <= 0 {
        return nil, ErrNotFound
    }
    // ...
    return &User{Name: "张三"}, nil
}

func main() {
    _, err := findUser(0)
    if errors.Is(err, ErrNotFound) {
        fmt.Println("用户不存在")
    }
}
```

## 9.7 panic 和 recover

### panic：程序崩溃

`panic` 用于**不可恢复的严重错误**，调用后程序会立即停止运行（先执行 defer，再退出）：

```go
func main() {
    fmt.Println("开始")
    panic("出了严重问题！")
    fmt.Println("这行不会执行")
}
// 输出：
// 开始
// panic: 出了严重问题！
// goroutine 1 [running]:
// ...
```

以下情况会自动 panic：
- 数组/切片越界访问
- 解引用 nil 指针
- 向已关闭的 channel 发送数据

### recover：从 panic 中恢复

`recover` 只能在 `defer` 函数中使用，用来捕获 panic：

```go
func safeFunction() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("捕获到 panic：", r)
        }
    }()

    fmt.Println("正常执行...")
    panic("boom!")
    fmt.Println("这行不会执行")
}

func main() {
    safeFunction()
    fmt.Println("程序继续运行") // 因为 recover 了，所以这行能执行
}
// 输出：
// 正常执行...
// 捕获到 panic：boom!
// 程序继续运行
```

### 什么时候用 panic？

| 场景 | 用 error | 用 panic |
|------|---------|---------|
| 文件不存在 | ✅ | ❌ |
| 网络请求失败 | ✅ | ❌ |
| 用户输入无效 | ✅ | ❌ |
| 配置文件格式错误 | ✅ | ❌ |
| 程序内部逻辑矛盾（不应该发生的情况） | ❌ | ✅ |
| 必须的初始化失败（如数据库连不上） | ❌ | ✅ |

**原则**：99% 的情况用 `error`，只有"程序无法继续运行"时才用 `panic`。

## 9.8 错误处理的最佳实践

### 1. 每个错误都要处理

```go
// 差：忽略错误（用 _ 接收）
data, _ := os.ReadFile("config.yaml")

// 好：检查并处理错误
data, err := os.ReadFile("config.yaml")
if err != nil {
    log.Fatal("读取配置失败：", err)
}
```

### 2. 添加上下文信息

```go
// 差：直接返回原始错误
return err

// 好：包装错误，添加上下文
return fmt.Errorf("创建用户失败: %w", err)
```

### 3. 在合适的层级处理错误

```go
// 底层函数：返回错误，不处理
func queryDB(sql string) ([]Row, error) {
    // ...
    return nil, fmt.Errorf("SQL 执行失败: %w", err)
}

// 中间层：包装错误，继续传递
func getUser(id int) (*User, error) {
    rows, err := queryDB("SELECT * FROM users WHERE id = ?")
    if err != nil {
        return nil, fmt.Errorf("获取用户 %d 失败: %w", id, err)
    }
    // ...
}

// 顶层（如 HTTP 处理器）：做最终处理
func handleGetUser(w http.ResponseWriter, r *http.Request) {
    user, err := getUser(123)
    if err != nil {
        log.Println("请求处理失败：", err)
        http.Error(w, "服务器内部错误", 500)
        return
    }
    // ...
}
```

## 9.9 综合实战：用户注册验证

```go
package main

import (
    "errors"
    "fmt"
    "strings"
)

type RegisterError struct {
    Field   string
    Message string
}

func (e *RegisterError) Error() string {
    return fmt.Sprintf("[%s] %s", e.Field, e.Message)
}

var (
    ErrUserExists = errors.New("用户已存在")
)

// 模拟已注册用户
var registeredUsers = map[string]bool{
    "admin":    true,
    "zhangsan": true,
}

func validateUsername(name string) error {
    if len(name) < 3 {
        return &RegisterError{Field: "username", Message: "用户名长度不能少于3个字符"}
    }
    if len(name) > 20 {
        return &RegisterError{Field: "username", Message: "用户名长度不能超过20个字符"}
    }
    if strings.Contains(name, " ") {
        return &RegisterError{Field: "username", Message: "用户名不能包含空格"}
    }
    return nil
}

func validatePassword(pwd string) error {
    if len(pwd) < 6 {
        return &RegisterError{Field: "password", Message: "密码长度不能少于6个字符"}
    }
    hasUpper := false
    hasDigit := false
    for _, ch := range pwd {
        if ch >= 'A' && ch <= 'Z' {
            hasUpper = true
        }
        if ch >= '0' && ch <= '9' {
            hasDigit = true
        }
    }
    if !hasUpper || !hasDigit {
        return &RegisterError{Field: "password", Message: "密码必须包含大写字母和数字"}
    }
    return nil
}

func register(username, password string) error {
    if err := validateUsername(username); err != nil {
        return fmt.Errorf("注册失败: %w", err)
    }
    if err := validatePassword(password); err != nil {
        return fmt.Errorf("注册失败: %w", err)
    }
    if registeredUsers[username] {
        return fmt.Errorf("注册失败: %w", ErrUserExists)
    }

    registeredUsers[username] = true
    fmt.Printf("用户 %s 注册成功！\n", username)
    return nil
}

func main() {
    testCases := []struct {
        username string
        password string
    }{
        {"ab", "Pass123"},          // 用户名太短
        {"newuser", "123"},         // 密码太短
        {"newuser", "abcdef"},      // 密码缺少大写和数字
        {"admin", "Pass123"},       // 用户已存在
        {"hainiu", "Hello123"},     // 成功
    }

    for _, tc := range testCases {
        fmt.Printf("\n尝试注册：用户名=%s 密码=%s\n", tc.username, tc.password)
        err := register(tc.username, tc.password)
        if err != nil {
            fmt.Println("错误：", err)

            var regErr *RegisterError
            if errors.As(err, &regErr) {
                fmt.Printf("  → 出错字段：%s\n", regErr.Field)
            }
            if errors.Is(err, ErrUserExists) {
                fmt.Println("  → 请换一个用户名")
            }
        }
    }
}
```

## 9.10 本章小结

| 概念 | 说明 |
|------|------|
| `error` 接口 | 只有一个 `Error() string` 方法 |
| 创建错误 | `errors.New("msg")` 或 `fmt.Errorf("...")` |
| 错误检查模式 | `if err != nil { return err }` |
| 自定义错误 | 定义 struct 并实现 `Error()` 方法 |
| 错误包装 | `fmt.Errorf("上下文: %w", err)` |
| 错误判断 | `errors.Is(err, target)` / `errors.As(err, &target)` |
| `panic` | 不可恢复的严重错误，程序崩溃 |
| `recover` | 在 defer 中捕获 panic |
| 最佳实践 | 每个错误都要处理、添加上下文、在合适层级处理 |

1. **基本错误处理**：写一个 `Sqrt(x float64) (float64, error)` 函数，当 x 为负数时返回错误
2. **自定义错误**：定义 `InsufficientBalanceError`（余额不足错误），包含当前余额和需要的金额，实现一个 `Withdraw` 函数
3. **错误包装**：模拟三层调用（数据层→服务层→控制器），每层包装错误并添加上下文，最终在控制器层打印完整的错误链
4. **recover 练习**：写一个 `SafeExecute(fn func()) error` 函数，安全地执行传入的函数，如果发生 panic 则返回 error 而不是崩溃
5. **综合题**：实现一个简单的配置文件解析器，读取 `key=value` 格式的配置，处理文件不存在、格式错误、值类型错误等多种异常情况
