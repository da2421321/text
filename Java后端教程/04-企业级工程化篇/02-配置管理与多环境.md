## 一、为什么需要多环境配置？

### 1.1 真实场景

一个项目至少有三个环境：

```
开发环境（dev）：你的电脑上
├── 数据库：localhost:3306/demo_dev
├── Redis：localhost:6379
└── 日志级别：DEBUG（尽可能详细）

测试环境（test）：公司测试服务器
├── 数据库：192.168.1.100:3306/demo_test
├── Redis：192.168.1.100:6379
└── 日志级别：INFO

生产环境（prod）：线上正式服务器
├── 数据库：db.mycompany.com:3306/demo_prod
├── Redis：redis.mycompany.com:6379
└── 日志级别：WARN（只记录警告和错误）
```

**不能把所有环境的配置混在一起**，否则上线时用了开发环境的数据库就出大事了。

## 二、Spring Boot 多环境配置

### 2.1 配置文件结构

```
resources/
├── application.yml          ← 公共配置（所有环境共用）
├── application-dev.yml      ← 开发环境独有配置
├── application-test.yml     ← 测试环境独有配置
└── application-prod.yml     ← 生产环境独有配置
```

### 2.2 公共配置 application.yml

```yaml
# 所有环境通用的配置
spring:
  application:
    name: demo-app
  profiles:
    active: dev              # 默认激活 dev 环境（本地开发用）
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: Asia/Shanghai
    default-property-inclusion: non_null

server:
  servlet:
    context-path: /api
  # 优雅关闭：收到关闭信号后等待正在处理的请求完成
  shutdown: graceful

  # 请求体大小限制
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 50MB
```

### 2.3 开发环境 application-dev.yml

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/demo_dev?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
  jpa:
    hibernate:
      ddl-auto: update       # 开发环境自动更新表结构
    show-sql: true            # 打印 SQL
  data:
    redis:
      host: localhost
      port: 6379

logging:
  level:
    root: info
    com.example.demo: debug   # 自己的代码打 DEBUG 日志
    org.hibernate.SQL: debug  # 打印 Hibernate SQL
```

### 2.4 生产环境 application-prod.yml

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://db.mycompany.com:3306/demo_prod?useSSL=true&serverTimezone=Asia/Shanghai
    username: ${DB_USERNAME}      # 从环境变量读取！不要硬编码
    password: ${DB_PASSWORD}      # 从环境变量读取！
  jpa:
    hibernate:
      ddl-auto: none              # 生产环境绝对不要自动改表！
    show-sql: false               # 关闭 SQL 打印
  data:
    redis:
      host: ${REDIS_HOST}
      port: 6379
      password: ${REDIS_PASSWORD}

logging:
  level:
    root: warn                    # 只记录警告和错误
    com.example.demo: info
```

### 2.5 如何切换环境

```bash
# 方式一：在 application.yml 中设置（适合本地开发）
spring:
  profiles:
    active: dev

# 方式二：启动时通过命令行参数指定（适合部署）
java -jar demo.jar --spring.profiles.active=prod

# 方式三：通过环境变量（适合 Docker/K8s）
export SPRING_PROFILES_ACTIVE=prod
java -jar demo.jar

# 方式四：在 IDEA 中设置
# Run Configuration → Environment variables → SPRING_PROFILES_ACTIVE=dev
```

## 三、配置绑定到 Java 类

### 3.1 为什么要绑定？

直接用 `@Value` 读取配置：

```java
@Value("${app.upload.max-size}")
private long maxSize;  // 散落在各处，难以管理
```

更好的方式：把一组相关的配置绑定到一个类上：

```yaml
# application.yml
app:
  upload:
    max-size: 10485760       # 10MB
    allowed-types: jpg,png,gif,webp
    save-path: /data/uploads
  jwt:
    secret: my-secret-key-at-least-32-chars-long!!
    expire-hours: 24
```

```java
package com.example.demo.config;

import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

import java.util.List;

@Data
@Component
@ConfigurationProperties(prefix = "app.upload")  // 绑定 app.upload 开头的配置
public class UploadProperties {
    private long maxSize;                // 对应 app.upload.max-size
    private List<String> allowedTypes;   // 对应 app.upload.allowed-types（自动分割逗号）
    private String savePath;             // 对应 app.upload.save-path
}
```

```java
@Data
@Component
@ConfigurationProperties(prefix = "app.jwt")
public class JwtProperties {
    private String secret;
    private int expireHours;
}
```

```java
// 使用：直接注入即可
@Service
@RequiredArgsConstructor
public class FileService {
    private final UploadProperties uploadProps; // 自动注入

    public void upload(MultipartFile file) {
        if (file.getSize() > uploadProps.getMaxSize()) {
            throw BizException.badRequest("文件大小超过限制: " + uploadProps.getMaxSize());
        }
        // ...
    }
}
```

## 四、配置优先级

Spring Boot 配置的加载优先级（从高到低）：

```
1. 命令行参数          --server.port=9090
2. 系统环境变量         SERVER_PORT=9090
3. application-{profile}.yml   （环境配置文件）
4. application.yml              （公共配置文件）
5. @ConfigurationProperties 默认值
```

**高优先级覆盖低优先级**。比如 application.yml 里写了 `port: 8080`，但启动时加了 `--server.port=9090`，最终端口就是 9090。

## 五、敏感信息管理

### 5.1 绝对不能做的事

```yaml
# ❌❌❌ 千万不要把密码写死在配置文件里提交到 Git！
spring:
  datasource:
    username: admin
    password: P@ssw0rd123!     # 这个被推到 GitHub 上就完了
```

### 5.2 正确做法

```yaml
# ✅ 方式一：使用环境变量（最常用）
spring:
  datasource:
    username: ${DB_USERNAME}     # 从环境变量 DB_USERNAME 读取
    password: ${DB_PASSWORD}     # 从环境变量 DB_PASSWORD 读取
```

```bash
# 部署时设置环境变量
export DB_USERNAME=admin
export DB_PASSWORD=P@ssw0rd123!
java -jar demo.jar
```

```yaml
# Docker Compose 中设置
services:
  app:
    environment:
      - DB_USERNAME=admin
      - DB_PASSWORD=P@ssw0rd123!
```

```yaml
# ✅ 方式二：使用 .env 文件（本地开发用）+ 加入 .gitignore
# 创建 .env 文件（不要提交到 Git！）
DB_USERNAME=root
DB_PASSWORD=123456

# 在 .gitignore 中添加
.env
```

### 5.3 .gitignore 安全清单

```gitignore
# 敏感文件
.env
*.key
*.pem
*-secret*
application-local.yml    # 本地个人配置
```

## 六、常用配置项速查

```yaml
# ========== 服务器 ==========
server:
  port: 8080
  servlet:
    context-path: /api
  shutdown: graceful                        # 优雅关闭
  tomcat:
    max-threads: 200                        # 最大线程数
    accept-count: 100                       # 等待队列

# ========== 数据库 ==========
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/demo
    username: root
    password: 123456
    hikari:                                 # 连接池配置
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000

# ========== JPA ==========
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    open-in-view: false                     # 关闭！避免懒加载问题

# ========== Redis ==========
  data:
    redis:
      host: localhost
      port: 6379
      password:
      timeout: 3000ms
      lettuce:
        pool:
          max-active: 20
          max-idle: 10
          min-idle: 5

# ========== Jackson (JSON序列化) ==========
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: Asia/Shanghai
    default-property-inclusion: non_null     # null 字段不输出
    serialization:
      write-dates-as-timestamps: false       # 日期不转时间戳

# ========== 日志 ==========
logging:
  level:
    root: info
  file:
    name: logs/app.log                       # 日志文件路径
  logback:
    rollingpolicy:
      max-file-size: 100MB                   # 单个日志文件最大
      max-history: 30                        # 保留天数
```

## 七、本章总结

| 要点 | 说明 |
|------|------|
| 多环境配置 | `application-{profile}.yml` 按环境分文件 |
| 切换环境 | `--spring.profiles.active=prod` |
| 配置绑定 | `@ConfigurationProperties` 绑定到类 |
| 敏感信息 | 用 `${ENV_VAR}` 环境变量，**不要写死密码** |
| 生产环境 | `ddl-auto: none`，`show-sql: false`，日志 WARN |
